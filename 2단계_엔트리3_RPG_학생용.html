🛠️ 오늘 쓸 도구 — 엔트리

엔트리봇은 지우고, "주인공"과 "적" 오브젝트를 추가해서 시작하자.
🎯 미션
마을과 전투, 두 개의 장면을 만들자.
⭐ 오늘의 새로운 개념 — 장면
RPG는 마을도 있고 전투도 있지? 한 화면에 다 넣지 않고, '장면'을 여러 개 만들어서 무대를 바꿔가며 진행할 수 있어!
📋 과제 — 이걸 해내면 성공!
①
장면 2개 추가
'마을' 장면과 '전투' 장면, 두 개를 만들기
②
각 장면에 배치
마을 장면엔 주인공, 전투 장면엔 주인공과 적을 놓기
💡 힌트 1 — 장면은 어디서 추가하지?
실행 화면 위쪽을 봐 — '장면'을 새로 추가하는 곳이 있어. '+'를 찾아봐!
💡 힌트 2 — 헷갈리지 않게!
중요! 같은 주인공이라도 마을 장면과 전투 장면에서 코드를 따로 짜야 해. 헷갈리지 않게!
🤫 약속
정답 블록은 안 알려줄게 — 직접 둘러보고 장면 추가하는 '+'를 네가 찾는 거야!
✅ 과제 체크리스트
✓
'마을' 장면을 만들었다
✓
'전투' 장면을 만들었다
✓
각 장면에 주인공·적을 배치했다
✓
위쪽 장면 탭을 눌러 두 장면이 잘 바뀌는지 확인했다
✏️ 이 블록이 뭐 하는 건지 말로 설명해보기
왜 마을과 전투를 한 화면에 다 넣지 않고 '장면'을 두 개로 나눴을까? 아빠한테 한 문장으로 설명해봐.
🆘 막혔을 때 — 3단계 규칙
① 10분 혼자 → 실행 화면 위쪽을 천천히 둘러보기
② AI에 힌트만 → "엔트리에서 무대(장면)를 새로 추가하는 곳은 어디야? 정답 말고 힌트만!"
③ 직접 만들고 설명 → 말로 설명 못 하면 다시!

단계 2
마을→전투 이동
적과 만나면 전투 무대로 넘어가게!
⏱ 15분
🎯 미션
마을에서 적과 만나면 전투 장면으로 넘어가게 하자.
📋 과제 — 이걸 해내면 성공!
①
주인공 움직이기
마을에서 주인공을 키보드로 움직이게
②
적에 닿았는지 확인
주인공이 적과 만났는지 알아채기
③
닿으면 전투 장면으로
만났으면 무대를 전투 장면으로 바꾸기
💡 힌트 1 — 닿았는지 확인은?
주인공이 적에 닿았는지 확인은 어느 카테고리였지?
(→ 판단 카테고리를 열어봐!)
💡 힌트 2 — 화면(장면) 바꾸기
화면(장면)을 바꾸는 블록을 찾아봐 — 어느 카테고리에 '장면'이라는 말이 있을까?
(→ 시작 카테고리!)
🤫 약속
블록 이름은 안 적어줄게. 카테고리 안을 직접 둘러보고 '장면'이라는 단어가 들어간 블록을 찾아봐!
✅ 과제 체크리스트
✓
마을에서 주인공이 움직인다
✓
주인공이 적에 닿았는지 확인한다
✓
닿으면 전투 장면으로 넘어간다
✓
▶ 실행해서 적과 만나면 전투 무대로 바뀌는지 확인했다
✏️ 이 블록이 뭐 하는 건지 말로 설명해보기
'적에 닿았는지' 확인하는 부분을 왜 '계속'으로 감쌌을까? 안 그러면 어떻게 될지 말로 설명해봐.
🆘 막혔을 때
블록을 못 찾겠으면 블록 모음 위쪽 검색창에 "장면", "닿", "이동" 같은 단어를 일부만 쳐봐.

단계 3
공격 함수
반복되는 공격을 '함수' 하나로 묶자!
⏱ 20분
🎯 미션 — 오늘의 핵심!
공격할 때마다 반복되는 동작을 '함수' 하나로 묶어보자.
⭐ 오늘의 새로운 개념 — 함수
공격할 때마다 똑같은 동작을 여러 번 쓰게 되지? 그 묶음에 '이름'을 붙여두면, 다음부터는 이름만 부르면 그 동작이 한꺼번에 실행돼.
이렇게 동작을 묶어 이름 붙인 걸 '함수'라고 불러!
📋 과제 — 이걸 해내면 성공!
①
적HP 변수 만들기
전투가 시작될 때 적HP를 50으로 정해두기
②
공격 동작을 함수로 묶기
공격 모습(모양 변화)과 적HP 줄이기를 한 묶음(함수)으로
③
키로 함수 부르기
스페이스 키를 누르면 그 함수를 부르기(호출)
💡 힌트 1 — 무엇을 찾아볼까?
공격할 때마다 똑같은 동작을 여러 번 쓰지? 이 묶음에 '이름'을 붙여두면 이름만 부르면 돼. 무엇을 찾아볼까? — 힌트: '함수'!
💡 힌트 2 — 만들기와 부르기는 따로!
함수는 '만들기(정의)'와 '부르기(호출)' 두 가지가 따로야.
만들기만 하고 안 부르면 아무 일도 안 일어나!
🤫 약속
정답 블록은 안 알려줄게 — '함수 만들기'를 찾아 직접 묶어봐.
✅ 과제 체크리스트
✓
'적HP' 변수를 만들고 전투 시작 때 50으로 정했다
✓
공격 동작(모습 변화 + 적HP 줄이기)을 함수로 묶었다
✓
스페이스 키로 그 함수를 부르게 했다
✓
▶ 실행해서 공격할 때마다 적HP가 줄어드는지 확인했다
✏️ 이 블록이 뭐 하는 건지 말로 설명해보기
'함수'를 왜 만들었을까? 함수로 묶지 않고 매번 똑같은 동작을 따로 쓰면 어떤 점이 불편할지 말로 설명해봐.
🆘 막혔을 때 — 3단계 규칙
① 10분 혼자 → "함수 만들기 → 안에 동작 넣기 → 키로 부르기" 순서를 종이에 그려보기
② AI에 힌트만 → "엔트리에서 반복되는 동작을 하나로 묶어 이름을 붙이려면 어떤 기능을 써야 해? 정답 말고 힌트만!"
③ 직접 만들고 설명 → 말로 설명 못 하면 다시!

단계 4
승리 판정·완성
적HP가 0이 되면 승리! 게임 완성!
⏱ 15분
🎯 미션
적HP가 0이 되면 '승리!'가 뜨고 게임이 멈추게 하자.
📋 과제 — 이걸 해내면 완성!
①
적HP를 계속 확인
적HP가 0 이하인지 멈추지 않고 살펴보기
②
0 이하면 승리 말하기
적HP가 다 닳으면 '승리!'라고 말하기
③
게임 멈추기
승리하면 게임이 끝나게(멈추게) 하기
💡 힌트 1 — 계속 확인하려면?
적HP가 0이 됐는지 계속 확인하려면 무엇으로 감쌀까?
(→ 흐름 카테고리!)
💡 힌트 2 — 0 이하인지 비교
적HP가 0 이하인지 비교하는 블록은 어디 있을까?
(→ 판단 카테고리!)
🤫 약속
정답 블록은 안 알려줄게 — 카테고리만 힌트야. 비교·멈춤 블록은 네가 찾는 거야!
