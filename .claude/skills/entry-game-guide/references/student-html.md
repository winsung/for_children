# 학생용 HTML 제작 상세

## 캐논 템플릿
**`2단계_엔트리1_별받기_학생용.html`** 을 복제해 내용만 교체한다. JS(`showPage`, `toggleCheck`, `updateProgress`, null-safe 완료 배너)와 CSS를 그대로 가져오고, 탭 내용만 바꾼다.

## 디자인 규칙
- 테마: 라이트 `#f0f4ff` 배경
- 폰트: Noto Sans KR
- 레이아웃: 탭형(상단 탭 + `showPage(n)`), 스크롤. iPad Safari용.
- 터치 친화: 최소 44px 터치 영역. `user-scalable=no`(핀치줌 방지).
- 단일 파일: CSS/JS 인라인, 외부 JS 라이브러리 금지.

## 동기화 (반드시 일치)
탭 개수 = `#page-N` 개수 = `#check-N` 개수 = `TOTAL_PAGES`. 게임당 보통 4탭(STEP 묶음). 어긋나면 진행률/완료 배너가 깨진다.

## STEP을 탭으로 묶기
강사용 STEP은 보통 5~6개, 학생용 탭은 4개다. STEP을 의미 단위로 ~4탭에 묶는다:
- **준비/세팅 STEP은 다음 STEP과 합친다** (예: STEP1 오브젝트 준비 + STEP2 시작 위치 → 탭1).
- 신개념이 등장하는 STEP은 **단독 탭**으로 둬서 충분히 다룬다 (예: 신호·함수·장면 도입 탭).
- 마지막 탭은 "완성/승리 판정"으로 마무리.
- 탭 제목은 아이가 보는 미션 단위로(예: "별 받기·완성"), STEP 번호를 그대로 쓰지 않는다.

## 절대 원칙 — 과제 + 힌트만, 정답 블록 금지
학생용은 **무엇을 할지(과제)와 어디서 찾을지(힌트)만** 준다. 완성 블록 시퀀스를 절대 적지 않는다. 이게 이 자료의 핵심 — 아이가 힌트만으로 스스로 블록을 찾아 조립해야 학습이 된다.

- ✅ 좋은 힌트: "키 입력 블록은 어느 카테고리에 있을까? (→ 시작)", "왼쪽으로 가려면 x좌표를 +로? -로? 둘 다 해보고 골라!", "예전 피하기 게임에서 폭탄을 어떻게 계속 만들었지? 떠올려봐."
- ❌ 금지: "x좌표를 -10 만큼 바꾸기", "복제본이 처음 생성되었을 때", 즉 강사용에 있는 블록 라인 그대로.
- 개념어(함수/장면/신호/변수/적HP 등)는 힌트에 써도 된다. 금지되는 건 **완성된 블록 라인**.

## 탭 구성 (각 탭)
- **미션** 한 줄 (`.mission`)
- **과제** 목록 (`.step-list`) — 달성 목표 (블록 아님)
- **힌트** (`.bubble`) — 카테고리·유도 질문. 보통 "정답 블록은 안 알려줄게" 한 줄 포함
- **체크리스트** (`.checklist`, `id="check-N"`) — 과제 항목 체크
- **"이 블록이 뭐 하는 건지 말로 설명해보기 ✏️"** 칸 (`.explain-box`)
- (선택) 막혔을 때 3단계 규칙 박스

## 마지막 탭 추가 요소
- 학습형 AI 질문 예시 (✅/❌ 대비) — "엔트리에서 …하려면 어떤 블록이 필요해? 정답 말고 힌트만!"
- (해당 시) 다음 단계 호기심 훅 — 예: "오늘 만든 '함수'는 파이썬에서도 똑같이 쓴대! 어떤 점이 닮았을까?"
- 완성 축하 배너 (`#complete-all`, 모든 체크 완료 시 표시)
