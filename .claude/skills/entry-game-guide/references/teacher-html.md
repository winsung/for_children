# 강사용 HTML 제작 상세

## 캐논 템플릿
**`2단계_엔트리1_별받기_강사용.html`** 을 복제해 내용만 교체한다. (게임②③ 강사용도 이걸 복제해 만들었다.) 처음부터 CSS를 짜지 말 것 — 디자인 일관성이 공짜로 유지된다.

## 디자인 규칙
- 테마: 다크 `#0f1117` 배경, 강조 Teal `#14b8a6`
- 폰트: Noto Sans KR + Fira Code(코드)
- 레이아웃: **스크롤·섹션·목차형** (발표 슬라이드 아님). 빌드를 옆 창에 띄우고 앞뒤로 스크롤하며 따라 만드는 용도.
- 단일 파일: CSS/JS 인라인, 외부 JS 라이브러리 금지, 구글 폰트 `<link>`만 허용.

## 컴포넌트 (템플릿에 이미 있음 — 그대로 재사용)
- `.wrap` 헤더 + `.toc` 목차
- `<section>` + `<h2>` (섹션별)
- `.step` / `.step-num` (STEP 카드)
- `.shot`(img + `.caption`) / `.shot-row` (스크린샷)
- `.box teal`(팁/신개념) / `.box amber`(주의) / `.box coral`(자주 막히는 곳)
- `.code` 코드 블록 + 토큰 클래스(`.c .k .f .n .s .cat`) — `[카테고리]` 라벨 + ┣/┗ 표기

## 섹션 구성 (순서)
1. 헤더 — 제목 "{단계} 엔트리 {N} — {게임} / 강사용 (Kay)"
2. 목차
3. 게임 소개 + 한 줄 규칙
4. 오브젝트 준비 — 오브젝트 추가/검색 스크린샷(`.shot`). 카테고리 위치는 `cat_*.png` `.shot-row`로
5. STEP별 블록 제작 — 각 STEP을 `.step` 카드 + `.code` 블록 표기로. **정답 블록을 그대로 보여준다.**
6. 신개념 포인트 (`.box teal`) — 이 게임에서 처음 나오는 블록/개념을 집중 설명
7. 자주 막히는 곳 (`.box coral`) — 증상 + 확인할 것
8. 3단계 도움 규칙 (`.box amber`) — (1)10분 혼자 (2)AI 힌트만 (3)직접 수정+말로 설명. 좋은 질문/나쁜 질문 예시 포함
9. 변형 아이디어 — 빨리 끝낸 아이를 위한 확장

## 스크린샷 참조
- 재사용 가능(이미 `assets/screenshots/`): `01_home.png`, `03_editor_areas.png`, `cat_{시작,흐름,움직임,생김새,판단,계산,자료}.png`
- 게임 신규: `assets/screenshots/entry/{게임폴더}/...png`
- 참조하는 모든 이미지는 디스크에 실제로 존재해야 한다. 약하거나 없는 스크린샷은 솔직한 캡션으로 위치를 안내하거나 img를 빼고 텍스트만 둔다 (가짜 금지).

## 파이썬 다리 (해당 시)
신개념이 텍스트 코딩으로 이어지면(특히 **함수 → 파이썬 `def`**) 전용 `.box teal` 섹션으로 명시한다. 파이썬 미리보기 스니펫을 넣을 땐 **실제로 실행되는 올바른 코드**여야 한다 — 예: 함수 안에서 모듈 변수를 재할당하면 `global` 없이는 `UnboundLocalError`. 파라미터+return 형태로 쓸 것.
