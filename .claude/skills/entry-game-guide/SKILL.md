# 엔트리 게임 가이드 제작

11세 아이를 위한 엔트리 블록코딩 게임의 **강사용 가이드(다크·스크린샷 포함)** 와 **학생용 자료(라이트·과제+힌트만)** 를 한 쌍으로 만든다. 이 프로젝트의 디자인·교육 철학(직접 타이핑, 정답 안 알려주기, 3단계 도움 규칙)을 자료에 녹인다.

## 핵심 원칙 (왜 이렇게 하는가)
- **강사용은 정답 블록을 보여주고, 학생용은 절대 안 보여준다.** 학생이 힌트만으로 스스로 블록을 찾아 조립해야 "왜 이렇게 동작하는지 말로 설명"하는 학습이 된다. 이게 프로젝트의 존재 이유라 양보 불가.
- **개념은 점진적으로.** 새 게임은 이전 게임이 다룬 개념을 재활용하고 신개념 1~2개만 얹는다. 한꺼번에 많이 넣으면 아이가 막힌다.
- **기존 산출물을 캐논 템플릿으로 복제·변형한다.** 처음부터 만들지 말 것. 검증된 파일을 베껴 내용만 갈아끼우면 디자인 일관성이 공짜로 유지된다.

## 입력 (시작 전 확정할 것)
이 정보가 없으면 사용자에게 먼저 물어 확정한다:
1. **게임 이름 + 한 줄 규칙** (예: "별 받기 — 좌우 키로 바구니를 움직여 떨어지는 별을 받자")
2. **오브젝트 목록** (플레이어/장애물/배경 등)
3. **신개념 1~2개** (예: 키보드 입력, 점수 변수) — 이전 게임과 겹치지 않는 새 개념
4. **STEP별 블록 흐름** (정답 블록 시퀀스). 강사용의 뼈대이자 학생용 힌트의 원본. 아래 블록 표기로 적는다.

## 블록 표기 규칙
강사용 코드 블록은 `[카테고리]` 라벨 + ┣/┗ 들여쓰기로 렌더링한다. 예:
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[흐름]   계속 반복하기
[움직임]   ┗ 마우스포인터 위치로 이동하기
```
자주 쓰는 카테고리: 시작/흐름/움직임/생김새/판단/계산/자료/함수. 조립된 블록은 **스크린샷이 아니라 이 표기로** 보여준다 (드래그 결과 자동 캡처는 불안정).

## 제작 순서

### 1. 스크린샷 (강사용에 들어갈 화면)
`scripts/capture-entry-games.js`(playwright)를 확장해 새 게임에 필요한 화면을 캡처한다. 자세한 방법·한계는 `references/screenshots.md` 참조.
- 핵심 한계: **로그아웃·빈 에디터로는 신호/장면/함수 등 깊은 UI가 안 잡힌다.** 약한 스크린샷은 가짜로 두지 말고 **솔직한 캡션**으로 실제 위치를 안내하거나 텍스트로 대체한다.
- 오브젝트 검색 화면은 잘 잡힌다. 카테고리 화면(`cat_*.png`)은 기존 것 재사용.

### 2. 강사용 HTML
`references/teacher-html.md`를 읽고 따른다. 요약:
- **캐논 템플릿: `2단계_엔트리1_별받기_강사용.html`** 을 복제해 내용만 교체.
- 다크(#0f1117), Teal(#14b8a6), Noto Sans KR + Fira Code, 스크롤·섹션·목차형(`.wrap`/`.toc`/`.step`/`.box teal|amber|coral`/`.code`/`.shot`).
- 섹션: 소개+한 줄 규칙 → 오브젝트 준비(스크린샷) → STEP별 블록 제작 → 신개념 포인트(teal) → 자주 막히는 곳(coral) → 3단계 도움 규칙(amber) → 변형 아이디어. **정답 블록을 보여준다.**

### 3. 학생용 HTML
`references/student-html.md`를 읽고 따른다. 요약:
- **캐논 템플릿: `2단계_엔트리1_별받기_학생용.html`** 을 복제해 내용만 교체.
- 라이트(#f0f4ff), 탭형(`showPage(n)`), `user-scalable=no`, `.checklist`(id `check-N`), 완료 배너(null-safe).
- 탭마다: 미션 + 과제 + **힌트(카테고리·유도 질문만, 정답 블록 금지)** + 체크리스트 + "말로 설명해보기" 칸. 마지막 탭에 학습형 AI 질문 예시 + 완성 배너.

### 4. 연결
- `index.html` 2단계 섹션에 강사용·학생용 카드 2개 추가 (기존 `.material` 패턴).
- `2단계_커리큘럼.md` 투트랙 표/신개념 표에 새 게임 반영.

## 파일명
`{단계}_엔트리{N}_{게임}_{대상}.html` (예: `2단계_엔트리1_별받기_강사용.html`)

## 검증 (커밋 전 필수)
게임별 핵심 블록 문자열을 정해 grep으로 확인한다. 상세는 `references/verification.md`.
- 학생용: `grep -c "<핵심 정답 블록들>" 학생용.html` → **0** (정답 누출 없음)
- 강사용: 같은 문자열 → **>0** (정답 블록 존재)
- 공통: 디자인 토큰(강사 #0f1117 / 학생 #f0f4ff), `script src="http"` 없음, 참조 스크린샷 모두 존재, 태그 균형.

## 게임 추가 시 회차/개념 연결
새 게임의 신개념은 다음 단계로 가는 다리가 되게 한다. 특히 **함수 → 파이썬 `def`** 처럼 3단계 텍스트 코딩으로 이어지는 연결을 강사용에 명시하면 좋다. 기존 3게임(별받기=키보드/변수, 두더지=신호/시간, RPG=장면/함수)을 참고해 겹치지 않는 신개념을 고른다.
