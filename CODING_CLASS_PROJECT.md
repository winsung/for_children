# AI 활용 어린이 코딩 클래스 — 프로젝트 히스토리

> **Claude Code 인수인계 문서**
> 이 파일은 claude.ai 대화에서 설계·제작된 내용을 Claude Code가 이어받아 작업할 수 있도록 작성되었습니다.

---

## 1. 프로젝트 개요

### 목적
11세 아이(초등 5학년)를 대상으로 부모(Kay)가 직접 진행하는 **홈클래스 AI 코딩 교육** 자료 제작

### 수업 환경
| 구분 | 환경 |
|------|------|
| 강사(Kay) | Windows PC → TV 화면 공유 |
| 학생(아이) | iPad + 키보드·트랙패드 |
| 강사 자료 | HTML 프레젠테이션 (PC 브라우저) |
| 학생 자료 | HTML 인터랙티브 (iPad Safari) |

### 핵심 교육 철학
- AI를 **"정답기계"가 아닌 "학습 파트너"** 로 사용
- **Claude Code / Cursor / Copilot 등 에이전트형 도구는 전 기간 사용 금지** (자동완성이 직접 타이핑·이해 기회를 빼앗음)
- 코드는 반드시 **직접 타이핑** (복붙 금지)
- 막혔을 때 → **3단계 도움 규칙** 적용 (아래 상세)
- 핵심 기준: **"이 코드가 왜 이렇게 동작하는지 말로 설명할 수 있는가?"**

## 2. 3단계 도움 요청 규칙 (수업 헌법)

모든 수업에서 일관되게 적용해야 하는 핵심 원칙.

```
Step 1 (필수)    → 10분 혼자 생각
                   오류 메시지 읽기, 코드 처음부터 다시 보기

Step 2 (조건부)  → AI에게 힌트만 요청
                   코드 전체 붙여넣기 금지
                   "답 말고 원인만 알려줘. 코드는 내가 고칠게"

Step 3 (필수)    → 직접 수정 + 말로 설명
                   설명 못 하면 이해 안 한 것 → 재도전
```

### AI 질문 패턴 — 금지 vs 권장

| ❌ 금지 (의존형) | ✅ 권장 (학습형) |
|----------------|----------------|
| "이 코드 고쳐줘" (전체 붙여넣기) | "NameError가 왜 나는지 원인만 알려줘" |
| "for문 예제 코드 짜줘" | "for문이 while문이랑 어떻게 달라?" |
| "숫자 맞히기 게임 만들어줘" | "필요한 기능 목록 알려줘. 코드는 내가 짤게" |
| "이 코드 맞아?" | "틀렸으면 왜 틀렸는지 이유만, 정답은 말하지 마" |

### 1회차: AI가 뭔지 물어보기 (60분)

**목표:** AI의 개념과 한계를 체험으로 이해

**진행 흐름:**
1. 도입 (10분) — "AI가 뭔지 알아?" 질문으로 시작
2. 활동1 (20분) — claude.ai 접속, 아래 질문 직접 입력
   - "AI가 뭐야? 초등학생에게 설명해줘"

   - "틀린 대답을 하기도 해?"
   - "너한테 물어보면 안 되는 게 있어?"
3. 활동2 (20분) — AI vs 나 비교 실험
   - "1부터 100까지 더하면?" → 속도 비교
   - "파란색이 예뻐, 빨간색이 예뻐?" → AI의 취향 한계 발견
4. 정리 (10분) — 노트에 "오늘 알게 된 것 1가지" 직접 쓰기

**부모 팁:** "AI가 다 해주면 되잖아요" → "AI한테 공원 놀러가달라고 해봐. 할 수 있을까?"

### 2회차: 컴퓨터처럼 생각하기 (60분)

**목표:** 알고리즘 = 순서대로 정확하게 지시하기 개념을 몸으로 체험

**핵심 설계 의도:**
이 수업은 **언플러그드(unplugged) 역할놀이** 방식이다.
Python 코드를 배우기 전에, "코딩 = 컴퓨터에게 정확한 순서로 지시하기"라는 감각을
컴퓨터 없이 몸으로 먼저 익히는 것이 목적.
나중에 IndentationError, SyntaxError를 만났을 때
"아, 컴퓨터한테 틀린 지시를 한 거구나"로 자연스럽게 연결됨.

**두 가지 개념을 동시에 가르치는 구조:**
1. **정확한 명령** — "앞으로 가" → 로봇이 "얼마나요?" 로 되물음
   → 컴퓨터는 모호한 지시를 스스로 해석하지 않는다
2. **순서 (절차적 사고)** — 순서가 바뀌면 결과도 바뀐다
   → 나중에 if/for 블록 순서 개념으로 연결

**진행 흐름:**

1. 도입 (10분) — **로봇 역할놀이**
   - Kay = 눈 감은 로봇, 아이 = 프로그래머
   - 아이가 지시하는 대로만 정확하게 움직이기
   - "앞으로 가" → "얼마나요?"
   - "오른쪽으로" → "몇 도요?"
   - "우산 들고 나가" → "문은 어떻게 해요? 열라는 명령이 없는데요"
   - **핵심 포인트:** 로봇이 일부러 순서를 어기는 것도 연기해보기
     아이가 "당연히 문을 열어야지"라고 느끼는 순간,
     "컴퓨터는 그 당연함을 모른다"는 것을 실감하게 됨

2. 활동1 (15분) — **알고리즘 카드 게임**
   - 종이 카드에 아침 루틴 순서 쓰기
   - "일어나기 → 세수 → 아침 먹기 → 학교 가기"
   - 순서를 섞으면 어떻게 되는지 웃으며 이야기하기

3. 활동2 (25분) — **엔트리 첫 접속 (playentry.org)**
   - 고양이 캐릭터 → 블록 드래그 → "앞으로 10 이동"
   - 목표: 캐릭터를 원하는 위치로 이동시키기
   - 블록 순서 바꿔보기 → 결과가 달라짐 관찰

4. 정리 (10분) — 노트에 "알고리즘이란 _____ 이다" 완성

**핵심 질문 (부모용):**
- "이 블록 순서를 바꾸면 어떻게 될 것 같아?" (실행 전에 예측하게 하기)
- "알고리즘이란 뭐라고 생각해?"

## 5. 2단계 상세 설계 (8회차: 5~12회차)

> **커리큘럼 문서(docx)에 정의됨. HTML 자료 미제작 상태.**

### 단계 목표
- Python 텍스트 코딩 직접 타이핑
- AI를 "선생님"으로 활용하는 루틴 정착
- 오류 메시지를 두려워하지 않는 습관
- 텍스트 게임 처음부터 끝까지 혼자 완성

### 회차별 요약

| 회차 | 주제 | 핵심 개념 | 실습 |
|------|------|-----------|------|
| 5회차 | print와 변수 | 변수 = 상자 비유 | 자기소개 출력 프로그램 |
| 6회차 | input() | 사용자 입력 받기 | 대화형 자기소개 프로그램 |
| 7회차 | 조건문 if/elif/else | 들여쓰기 규칙 중요 | 숫자 맞히기 게임 v1 |
| 8회차 | 반복문 for/while | for vs while 차이 | 숫자 맞히기 게임 v2 |
| 9회차 | 디버깅 전문 수업 | 오류 4총사 (Syntax/Name/Indent/Type) | 의도적 버그 고치기 게임 |
| 10회차 | 함수 def | 함수 = 레시피 비유 | 계산기 함수 만들기 |
| 11회차 | 리스트 | 리스트 = 순서 있는 상자 모음 | 할 일 목록 프로그램 |
| 12회차 | 미니 프로젝트 | 종합 실습 | 텍스트 RPG 게임 완성 |

### 환경 세팅 (5회차 시작 전 필수)
- Pickcode (pickcode.io) 계정 생성
- Python 새 프로젝트 생성
- 첫 코드: `print("Hello, 세계!")`

### 자주 나오는 오류 4총사 (9회차 핵심)

| 오류 | 원인 | 해결 |
|------|------|------|
| SyntaxError | 오타, 따옴표/괄호 빠짐 | 빨간 줄 위치부터 확인 |
| NameError | 변수 이름 오타, 정의 전 사용 | 변수명 스펠링 확인 |
| IndentationError | 들여쓰기 칸 수 불일치 | Tab vs Space 혼용 금지 |
| TypeError | 다른 타입 연산 (문자+숫자) | str() 또는 int() 사용 |

## 6. 3단계 상세 설계 (8회차: 13~20회차)

> **커리큘럼 문서(docx)에 정의됨. HTML 자료 미제작 상태.**

### 단계 목표
- 딕셔너리·파일 입출력으로 실용적인 앱 제작
- AI API 최초 호출 ("내 코드가 AI를 쓴다" 경험)
- 나만의 챗봇 기획·개발·발표 완결

### 회차별 요약

| 회차 | 주제 | 핵심 개념 | 실습 |
|------|------|-----------|------|
| 13회차 | 딕셔너리 | 키-값 쌍 데이터 | 성적 관리 프로그램 |
| 14회차 | 파일 입출력 | open/read/write | 일기 앱 |
| 15회차 | 미니 앱 | 종합 실습 | 관심 주제 퀴즈 프로그램 |
| 16회차 | AI API 입문 | Anthropic API 호출 | 파이썬으로 Claude 호출 |
| 17회차 | 챗봇 기획 | 기획서 → 개발 사이클 | 종이 기획서 작성 |
| 18회차 | 챗봇 개발 1 | messages 리스트 | 챗봇 기본 구조 |
| 19회차 | 챗봇 개발 2 | 시스템 프롬프트 | 심화 기능 추가 |
| 20회차 | 최종 발표 | 포트폴리오 시작 | 챗봇 시연 + 발표 |

### API 사용 시 주의사항 (16회차)
```python
# API 키는 절대 코드에 직접 쓰지 않기
# 환경변수(os.environ) 사용
import anthropic
client = anthropic.Anthropic()  # ANTHROPIC_API_KEY 환경변수에서 읽음

message = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=200,
    messages=[{"role": "user", "content": question}]
)
print(message.content[0].text)
```

## 7. 4단계 — 자유 프로젝트

### 트랙 선택지

| 트랙 | 주요 기술 | 예시 |
|------|-----------|------|
| 🎮 게임 개발 | Pygame | 나만의 점프/슈팅 게임 |
| 🌐 웹 창작 | HTML/CSS + Flask | 좋아하는 주제 미니 웹사이트 |
| 📊 데이터 탐구 | Pandas, Matplotlib | 좋아하는 게임·스포츠 통계 분석 |
| 🤖 AI 앱 | Anthropic API | 나만의 특화 AI 도구 |
| 🔧 자동화 | Python 스크립트 | 숙제 도우미, 일정 알리미 |

### 자유 프로젝트 3원칙
1. **기획서 먼저** — 코딩 전 "무엇을 왜 만드는지" 종이에 쓰기
2. **AI는 파트너** — 코드 받지 않기. 설명 못 하면 사용 금지
3. **발표로 마무리** — 완성도보다 설명 능력이 목표
