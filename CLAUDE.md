# for_children 프로젝트

## 개요
11세 아이(초등 5학년, 2016년생)를 대상으로 부모(Kay)가 직접 진행하는 **홈클래스 AI 코딩 교육** 자료 프로젝트

## 수업 환경
- 강사(Kay): Windows PC → TV 화면 공유
- 학생(아이): iPad + 키보드·트랙패드
- 강사 자료: HTML 프레젠테이션 (PC 브라우저)
- 학생 자료: HTML 인터랙티브 (iPad Safari)

## 핵심 교육 철학
- AI는 "정답기계"가 아닌 "학습 파트너"
- **에이전트형 도구(Claude Code/Cursor/Copilot) 전 기간 사용 금지** (아이 학습 시)
- 코드는 반드시 **직접 타이핑** (복붙 금지)
- 3단계 도움 규칙: (1) 10분 혼자 생각 → (2) AI에게 힌트만 요청 → (3) 직접 수정 + 말로 설명
- 핵심 기준: "이 코드가 왜 이렇게 동작하는지 말로 설명할 수 있는가?"

## 커리큘럼 (4단계 로드맵)
| 단계 | 주제 | 회차 | 도구 | 상태 |
|------|------|------|------|------|
| 1단계 | AI와 친해지기 | 1~4회차 | Claude AI, 엔트리, 이미지AI | ✅ HTML 완료 |
| 2단계 | AI와 함께 만들기 | 5~12회차 | **엔트리(주력) + Python(Pickcode) 투트랙**, OpenClaw (집 안 Gemma) | ✅ HTML 완료 (엔트리 게임 3종 포함) |
| 3단계 | AI 활용 프로그래머 | 13~20회차 | Python, Anthropic API | ❌ HTML 미제작 |
| 4단계 | 자유 프로젝트 | 무제한 | 관심사 기반 선택 | — |

## HTML 자료 디자인 규칙

### 강사용 (PC→TV)
- 테마: 다크 (#0f1117 배경) — **두 포맷 모두 다크 유지**
- 폰트: Noto Sans KR + Fira Code (코드 블록)
- 레이아웃 (목적에 따라 2종):
  - **발표 슬라이드형** (수업 중 TV 공유용): 전체화면, 키보드 ← → 이동, 타이머(T키), 진행 바, 슬라이드 카운터. 예: `2단계_강사용.html`
  - **빌드 따라하기 스크롤·섹션형** (강사 선행/제작 가이드): 목차 + 섹션 + STEP + 스크린샷(`.shot`) + 콜아웃(`.box teal/amber/coral`). 도구를 옆 창에 띄우고 앞뒤로 스크롤하며 따라 만드는 용도. 예: `엔트리_선행가이드.html`, `2단계_엔트리*_강사용.html`
  - ⚠️ "이전과 같이" 류 요청이 오면 **가리키는 기존 파일을 먼저 열어** 포맷을 확인할 것 (일반 규칙과 실제 산출물이 어긋날 수 있음)
- 색상 계열: 1단계 Purple(#8b5cf6) / 2단계 Teal(#14b8a6) / 3단계 Amber(#f59e0b) / 4단계 Blue(#60a5fa)

### 학생용 (iPad)
- 테마: 라이트 (#f0f4ff 배경)
- 폰트: Noto Sans KR
- 레이아웃: 탭형 (회차별 탭 전환, 스크롤)
- UI: 터치 친화적 (최소 44px 터치 영역)
- 기능: 체크리스트, 진행률, 외부 링크, 완료 배너
- user-scalable=no (iPad 핀치줌 방지)
- **엔트리 게임 학생용은 "과제 + 힌트만"** — 정답 블록 시퀀스 금지(카테고리 힌트·유도 질문만). 제작/검토 시 핵심 블록 문자열로 `grep -c` = 0 확인 (강사용은 반대로 정답 블록 present 확인)

### 공통 원칙
- 구글 폰트 CDN 로드 (인터넷 필수)
- 외부 JS 라이브러리 사용 안 함 (순수 HTML/CSS/JS)
- 단일 파일로 완결 (CSS, JS 모두 인라인)

## 파일명 규칙
- 기본: `{단계}_{대상}.html` (예: 1단계_강사용.html, 1단계_학생용.html)
- 엔트리 게임: `{단계}_엔트리{N}_{게임}_{대상}.html` (예: 2단계_엔트리1_별받기_강사용.html)

## 엔트리 자료 · 스크린샷
- 엔트리 게임 가이드는 playentry.org 실제 화면 스크린샷을 포함 (강사용)
- 캡처: `scripts/capture-entry*.js` (playwright, headless). 결과 → `assets/screenshots/` (게임별: `assets/screenshots/entry/{star,mole,rpg}/`)
- ⚠️ **로그아웃·빈 에디터(`playentry.org/ws/`)로는 신호 만들기/장면 추가/함수 만들기 등 깊은 UI가 안 잡힘** (오브젝트 선택·로그인 필요). 보강하려면 Kay 로그인 세션 + 오브젝트 선택 후 캡처. 자동 캡처가 약하면 가짜 대신 솔직한 캡션으로 위치 안내
- 조립된 블록 코드는 자동 캡처하지 않고 HTML 내 블록 표기(`[카테고리]` + ┣/┗)로 렌더링
