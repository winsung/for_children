# 1단계 4회차 — 엔트리 선행 가이드 (강사용)

> **대상:** Kay (강사 본인)
> **목적:** 수업 전 직접 한번 만들어보면서 엔트리 사용법을 익히고, 막힐 만한 지점을 미리 파악하기
> **선행 소요 시간:** 약 60~90분 (편하게 따라가면 60분, 변형 실험까지 하면 90분)
> **만들 작품:** 고양이 피하기 게임

---

## 0. 선행 체크리스트 (수업 전날까지)

- [ ] 엔트리 회원가입 + 로그인
- [ ] 빈 작품 하나 만들어서 저장 → 다시 열기 성공
- [ ] 본 가이드를 따라 "고양이 피하기" 한번 완성
- [ ] 자녀가 "왜 이게 안 움직여요?" 라고 물어봤을 때 즉답 가능한 수준까지 디버깅 섹션 숙지
- [ ] (선택) 같은 게임을 처음부터 한 번 더 만들어보기 — 손에 익으면 수업 중 코칭이 자연스러워짐

---

## 1. 엔트리란?

- 한국과학창의재단(KOFAC)에서 만든 무료 블록코딩 교육 플랫폼
- 스크래치(Scratch)와 비슷하지만 **한국어 인터페이스** + 한국 교육과정 친화적
- 웹 기반: 설치 없이 브라우저에서 바로 사용 (오프라인 버전도 있음)
- URL: **https://playentry.org/** (구 entry.org.kr 에서 리다이렉트됨)

### 왜 4회차에 엔트리를 쓰는가?
- 1~3회차에서 "AI에게 묻기"로 사고력을 키웠다면, 4회차는 **실제 작품을 완성하는 성취 경험**
- 텍스트 코딩(2단계 Python) 들어가기 전 **조건문/반복문 개념을 시각적으로** 먼저 익힘
- 블록을 직접 끌어다 조립하므로 "복붙 금지" 원칙과 자연스럽게 맞물림

## 2. 회원가입 & 첫 진입

### 2.1 회원가입
1. https://playentry.org/ 접속
2. 우측 상단 **"회원가입"** 클릭
3. 14세 미만은 보호자 동의 필요 → **수업에서는 Kay 계정으로 로그인한 상태에서 자녀가 사용** 권장
4. 이메일/아이디 + 비밀번호 입력 → 가입

> **수업 운영 팁:** 자녀에게 별도 계정을 만들어주면 작품 누적되는 재미가 있음. 다만 4회차는 "처음 체험"이므로 Kay 계정으로 빠르게 진입해도 무방.

### 2.2 작품 만들기 진입
1. 로그인 후 상단 메뉴에서 **"만들기"** → **"작품 만들기"** 클릭
2. 새 빈 프로젝트가 열림 (기본 오브젝트: 엔트리봇)

---

## 3. 엔트리 화면 구조 (한눈에)

처음 작품 만들기 화면을 열면 아래처럼 4개 영역으로 나뉜다.

```
┌─────────────────────────────────────────────────────────┐
│  [상단 메뉴: 저장 / 되돌리기 / 작품 이름]                │
├──────────────────┬──────────────────┬───────────────────┤
│                  │                  │                   │
│   ① 실행 화면    │   ③ 블록 모음   │   ④ 블록 조립소  │
│   (장면)         │   (카테고리)    │   (코드 영역)    │
│                  │                  │                   │
│                  ├──────────────────┤                   │
│                  │                  │                   │
│  ② 오브젝트 목록 │                  │                   │
│   (좌측 하단)    │                  │                   │
└──────────────────┴──────────────────┴───────────────────┘
```

| 영역 | 이름 | 하는 일 |
|------|------|---------|
| ① | **실행 화면 (장면)** | 좌측 상단. ▶ 시작하기 버튼 누르면 여기서 게임이 돌아감 |
| ② | **오브젝트 목록** | 좌측 하단. 캐릭터/배경/장애물 등 모든 등장요소 리스트 |
| ③ | **블록 모음** | 중앙. 카테고리별로 블록이 정리되어 있음 (시작/흐름/움직임 등) |
| ④ | **블록 조립소** | 우측. ③에서 블록을 **끌어다(드래그) 놓는** 코드 영역 |

### 핵심 동작
- **블록 조립:** 블록 모음(③)에서 원하는 블록을 잡고 우측(④)으로 드래그 → 자석처럼 위아래로 붙음
- **블록 분리:** 블록을 잡고 빈 곳으로 드래그 → 떨어짐. 휴지통(④ 우측 상단)에 끌면 삭제
- **오브젝트 선택:** ②에서 오브젝트를 클릭하면 ④의 코드가 그 오브젝트의 코드로 바뀜
  - **⚠️ 가장 흔한 실수:** 캐릭터 코드를 짜는 줄 알았는데 장애물 코드를 짜고 있는 경우 — 항상 ② 좌측 하단에서 **어떤 오브젝트가 선택되어 있는지** 먼저 확인!

## 4. 블록 카테고리 빠른 개요

| 카테고리 | 색 | 자주 쓰는 블록 (4회차 기준) |
|----------|-----|------------------------------|
| **시작** | 초록 | `▶ 시작하기 버튼을 클릭했을 때` / `복제본이 처음 생성되었을 때` |
| **흐름** | 주황 | `계속 반복하기` / `만일 ~ 라면` / `~초 기다리기` / `자신의 복제본 만들기` / `복제본 삭제하기` / `모든 코드 멈추기` |
| **움직임** | 파랑 | `x좌표를 ~만큼 바꾸기` / `y좌표를 ~만큼 바꾸기` / `~ 위치로 이동하기` / `x:~ y:~ 위치로 이동하기` |
| **생김새** | 보라 | `~ 말하기` / `~ 모양으로 바꾸기` |
| **판단** | 진녹 | `~에 닿았는가?` / `마우스 포인터` |
| **계산** | 연두 | `0부터 10 사이의 무작위 수` |
| **자료** | 빨강 | (선택) `변수 만들기` → `점수` 등 |

> **외울 필요 없음.** 이 표는 "이 블록이 어디 카테고리지?" 막혔을 때만 보면 됨.

## 5. 고양이 피하기 게임 — 단계별 제작

### 게임 규칙 (자녀에게 설명할 한 줄)
> "고양이를 마우스로 움직여서 위에서 떨어지는 폭탄을 피하는 게임이야. 닿으면 게임 오버!"

### 사용할 오브젝트
1. **고양이** (플레이어 — 마우스 따라다님)
2. **폭탄** (장애물 — 위에서 떨어지며 복제됨)
3. **배경** (선택)

---

### STEP 1. 오브젝트 준비 (5분)

#### 1-1. 기본 엔트리봇 삭제
- 좌측 하단 ② 오브젝트 목록에서 "엔트리봇" 위에 마우스 올리기 → **X 버튼** 클릭

#### 1-2. 고양이 추가
1. 오브젝트 목록 위쪽 **"+ 오브젝트 추가하기"** 버튼 클릭
2. 카테고리에서 **"동물"** 선택 → **"고양이"** 또는 비슷한 캐릭터 선택
3. **"추가하기"** 클릭

#### 1-3. 폭탄(장애물) 추가
1. 다시 **"+ 오브젝트 추가하기"** 클릭
2. 검색창에 **"폭탄"** 또는 **"운석"** 입력 → 선택 → 추가
   - 없으면 **"공"**, **"별"** 등 무엇이든 OK

#### 1-4. (선택) 배경 추가
1. **"+ 오브젝트 추가하기"** → **"배경"** 탭 → 원하는 배경 선택

> **저장:** 상단 우측 **"저장하기"** 클릭. 작품 이름은 "고양이 피하기" 로.
> 한번 저장해두면 이후 자동저장도 안전함.

---

### STEP 2. 고양이가 마우스를 따라다니게 (10분)

**오브젝트 목록 ②에서 "고양이" 클릭** (반드시 먼저!)

블록 모음에서 다음을 우측 코드 영역으로 드래그 — 위에서 아래로 붙이기:

```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[흐름]   계속 반복하기
[움직임]   ┗ 마우스포인터 위치로 이동하기
```

**테스트:** 화면 좌측 상단 **▶ 시작하기** 클릭 → 마우스를 실행 화면 위에서 움직이면 고양이가 따라옴.

> **자주 막히는 곳:**
> - "마우스포인터 위치로 이동하기" 블록이 안 보임 → 카테고리 **움직임**에서 찾기. "x: y: 위치로 이동" 블록과 다른 블록이니 주의.
> - 고양이가 안 움직임 → ② 오브젝트가 "고양이"로 선택되어 있는지 확인. 다른 오브젝트에 코드를 붙였을 가능성 큼.

---

### STEP 3. 폭탄이 계속 복제되며 위에서 등장 (15분)

**② 오브젝트 목록에서 "폭탄" 클릭**

#### 3-1. 폭탄 본체: 1초마다 자기 자신 복제
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[움직임]   x: 0  y: 200  위치로 이동하기   (시작 위치를 화면 밖 위로)
[생김새]   모양 숨기기   (원본은 안 보이게)
[흐름]   계속 반복하기
[흐름]     ┣ 자신의 복제본 만들기
[흐름]     ┗ 1 초 기다리기
```

> **포인트:** 원본 폭탄은 화면 밖에서 숨어있고, 복제본만 화면에 떨어진다. 그래야 깔끔하게 관리됨.

#### 3-2. 복제본 동작: 무작위 위치에서 떨어지기
같은 폭탄 오브젝트의 코드 영역에 **새 시작 블록**으로 별도 흐름을 추가:

```
[시작]   복제본이 처음 생성되었을 때
[생김새]   모양 보이기
[움직임]   x: (-200 부터 200 사이의 무작위 수)  y: 200  위치로 이동하기
[흐름]   계속 반복하기
[움직임]     ┣ y좌표를 -5 만큼 바꾸기
[흐름]     ┗ 만일 ⟨ y좌표 < -130 ⟩ 라면
[흐름]         ┗ 이 복제본 삭제하기
```

- `(-200 부터 200 사이의 무작위 수)` → **계산** 카테고리
- `y좌표 < -130` → **판단**의 비교 블록 + **계산**의 `y좌표` 가져오기 (실행화면 세로 범위는 보통 -135 ~ 135)
- `이 복제본 삭제하기` → **흐름** 카테고리

**테스트:** ▶ 시작하기 → 폭탄들이 위에서 무작위 위치에 나타나 아래로 떨어짐.

> **자주 막히는 곳:**
> - 폭탄이 한곳에서만 떨어짐 → "무작위 수" 블록을 안 끼웠거나 시작 x를 고정값으로 둠
> - 폭탄이 끝없이 쌓여서 느려짐 → "이 복제본 삭제하기" 누락. 화면 아래로 사라지면 반드시 삭제해야 함
> - 원본 폭탄도 같이 떨어짐 → "모양 숨기기" 빠뜨림

---

### STEP 4. 충돌 시 게임 오버 (10분)

**② 오브젝트 목록에서 "고양이" 클릭** (다시 고양이로!)

기존 "마우스 따라가기" 코드는 그대로 두고, **새 시작 블록**으로 충돌 감지 흐름 추가:

```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[흐름]   계속 반복하기
[흐름]     ┗ 만일 ⟨ 폭탄 에 닿았는가? ⟩ 라면
[생김새]         ┣ "게임 오버!" 을(를) 말하기
[흐름]         ┗ 모든 코드 멈추기
```

- `폭탄에 닿았는가?` → **판단** 카테고리에서 `마우스포인터에 닿았는가?` 블록의 드롭다운을 **"폭탄"** 으로 변경

**테스트:** 일부러 폭탄에 부딪쳐보기 → "게임 오버!" 말풍선 + 모든 움직임 정지.

---

### STEP 5. (선택, 시간 남으면) 점수/시간 변수 추가 (10분)

#### 5-1. 변수 만들기
1. **자료** 카테고리 → **변수 만들기** 클릭
2. 변수 이름: **"버틴 시간"** → 확인

#### 5-2. 1초마다 +1 (아무 오브젝트에 붙여도 됨, 보통 고양이에)
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[자료]   버틴 시간 을 0 (으)로 정하기
[흐름]   계속 반복하기
[자료]     ┣ 버틴 시간 에 1 만큼 더하기
[흐름]     ┗ 1 초 기다리기
```

화면 좌측 상단에 변수 박스가 자동 표시됨.

## 6. 자주 막히는 곳 — 디버깅 체크리스트

수업 중 자녀가 "안 돼요"라고 할 때 순서대로 확인:

| 증상 | 확인할 것 |
|------|-----------|
| 블록이 우측에 안 붙음 | 블록을 살짝 위/아래로 흔들어 자석처럼 붙는 위치까지 끌어가기 |
| 코드 짰는데 아무 반응 없음 | ① ▶ 시작하기 버튼을 눌렀는지 ② 시작 블록(`▶ 시작하기 버튼을 클릭했을 때`)이 맨 위에 있는지 |
| 캐릭터가 안 움직임 | ② 오브젝트 목록에서 **올바른 오브젝트**가 선택되어 있는지 (다른 오브젝트에 코드 짠 경우 많음) |
| 폭탄이 한 자리에서만 떨어짐 | "무작위 수" 블록이 시작 x 위치에 끼워졌는지 |
| 복제본이 끝없이 늘어남 / 화면이 느려짐 | "이 복제본 삭제하기" 누락 — 화면 아래 도달 시 삭제 조건 확인 |
| 게임이 안 끝남 (충돌해도 계속됨) | "닿았는가?" 블록의 대상이 **폭탄**인지 확인 (드롭다운 잘못 둔 경우) |
| 모든 게 두 번씩 일어남 | 시작 블록이 두 개 있는데 같은 동작을 함 — 코드 영역 스크롤해서 확인 |

## 7. AI 도움 흐름 — 4회차 적용

CLAUDE.md의 **3단계 도움 규칙** 그대로 적용:

1. **10분 혼자 생각** — "이 블록이 뭘 해야 할까?"
2. **AI에게 힌트만 요청** — 학생용에서 가르친 학습형 질문 그대로
   - 좋은 예: "엔트리에서 오브젝트가 마우스를 따라다니게 하려면 어떤 블록을 써야 해? 정답 말고 힌트만."
   - 나쁜 예: "엔트리 고양이 피하기 게임 코드 알려줘"
3. **직접 수정 + 말로 설명** — 완성 후 "이 블록이 뭘 하는 거야?" 물어보기

> **수업 중 Kay 역할:** 자녀가 막혀도 즉답하지 않기. "어디서 막혔는지 말로 설명해봐" → "그럼 어떤 블록이 필요할 것 같아?" 순서로 유도.

---

## 8. 수업 진행 타임라인 (90분)

| 시간 | 활동 | 비고 |
|------|------|------|
| 0~5분 | 도입 — "오늘은 진짜 게임을 만든다" | 엔트리 화면 같이 보기 |
| 5~15분 | AI에게 아이디어 요청 + 게임 규칙 정하기 | "고양이 피하기" 베이스로 변형 OK (예: 별 받기) |
| 15~25분 | STEP 1~2 (오브젝트 준비 + 마우스 따라가기) | 첫 성공 경험 빨리 만들기 |
| 25~50분 | STEP 3 (폭탄 복제) | 가장 어려운 구간 — 함께 천천히 |
| 50~65분 | STEP 4 (충돌 감지) | "닿았는가?" 블록 처음 등장 |
| 65~75분 | STEP 5 또는 자유 변형 | 시간/점수 또는 폭탄 속도 바꾸기 |
| 75~85분 | 발표 — "이 블록이 뭐 하는 거야?" | 핵심 블록 3개 설명 |
| 85~90분 | 노트 정리 + 칭찬 | 완성한 것 무조건 칭찬 |

## 9. 변형 아이디어 (자녀가 빨리 끝나거나 더 만들고 싶어할 때)

- **별 모으기 모드:** 폭탄 대신 별. 닿으면 점수+1 (이 복제본 삭제)
- **속도 점점 빨라지기:** 시간 변수가 늘어날수록 폭탄 떨어지는 간격을 짧게
- **목숨 시스템:** 변수 "목숨" = 3 → 닿으면 -1, 0이 되면 게임 오버
- **여러 종류 장애물:** 폭탄 + 운석 두 종류

이런 변형도 **AI에게 "어떤 블록을 추가해야 할까?"** 로 묻게 유도.
