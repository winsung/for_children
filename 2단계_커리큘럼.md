## 투트랙 운영 구조 (엔트리 주력 + 파이썬 병행)

11세 아이가 아직 파이썬 텍스트 코딩에 부담을 느껴, **엔트리(블록코딩)를 주력**으로 더 진행하고 **파이썬은 맛보기로 가볍게 병행**한다. 엔트리 게임 1개당 1~2회차를 쓰고, 그 사이에 파이썬을 1회차씩 끼운다. 주 1~2회 운영이라 회차는 유동적 — 아이 속도에 맞춰 엔트리를 늘리거나 파이썬을 줄여도 된다. 아래는 기본 리듬이며 절대값이 아니다.

| 회차 | 트랙 | 내용 |
|------|------|------|
| 5 | 🐍 Python | (기존) Pickcode 입문 — Hello / 변수 맛보기 |
| 6~7 | 🧩 Entry ① | **별 받기** — 키보드 조종 + 점수 변수 |
| 8 | 🐍 Python | 조건문 / 입력 가볍게 (엔트리 ①의 조건 개념과 연결) |
| 9~10 | 🧩 Entry ② | **두더지 잡기** — 신호(방송하기) + 변수/타이머 |
| 11 | 🐍 Python | 반복문 가볍게 |
| 12 | 🧩 Entry ③ | **간단 RPG** — 장면 전환 + 함수 |

### 엔트리 3게임 신개념 (점진 설계)

피하기 게임(1단계)이 이미 다룬 반복·복제·조건·충돌·변수 위에, 게임마다 신개념 1~2개를 얹는다.

| 게임 | 신개념 | 파이썬 연결 |
|------|--------|-------------|
| ① 별 받기 | 키보드 입력, 점수 변수 | 변수·조건 |
| ② 두더지 잡기 | 신호 보내기/받았을 때, 시간 변수 | 이벤트·상태 |
| ③ 간단 RPG | 장면 전환, **함수** | **함수 = 파이썬 `def`** — 3단계로 가는 다리 |

> 게임별 자료: `2단계_엔트리{N}_{게임}_강사용.html`(다크·스크린샷 포함) / `_학생용.html`(과제·힌트만). 강사용은 정답 블록을 보여주고, 학생용은 정답 블록 없이 힌트만 제공한다.

### 학습 목표

- `print()` 함수로 화면에 출력하기
- 변수의 개념(상자 비유)과 사용법 이해하기
- 오류를 일부러 만들어보고 메시지 읽는 연습하기

### 핵심 개념 설명

#### print() — 화면에 출력하기

```python
print("Hello, 세계!")
print("안녕하세요")
```

**부모 설명 방법:** "print는 컴퓨터한테 '이걸 화면에 보여줘'라고 시키는 거야"

#### 변수 — 상자 비유

```python
name = "철수"       # "name" 이라는 상자에 "철수" 를 담는다
age = 11            # "age" 라는 상자에 11 을 담는다

print(name)         # 상자 안에 뭐가 들었는지 꺼내 보여준다
print(age)
```

**부모 설명 방법:** "변수는 물건을 담아두는 상자야. 상자에 이름을 붙이면 나중에 꺼내 쓸 수 있어"

#### 문자열 + 숫자 조합

```python
name = "철수"
age = 11

print("안녕! 나는 " + name + " 이고 " + str(age) + "살이야")
```

> ⚠️ **중요:** 숫자(age)를 문자열과 합칠 때는 `str()` 로 감싸야 한다.
> 안 하면 TypeError 발생 → 오류 체험의 기회로 활용

## 6회차: input()과 대화형 프로그램

**소요 시간:** 60분

### 학습 목표

- `input()` 함수로 사용자 입력 받기
- 변수와 input을 조합한 대화 프로그램 만들기
- `int()` 로 문자열을 숫자로 변환하기

### 핵심 개념 설명

#### input() — 사용자가 타이핑할 수 있게 하기

```python
name = input("이름이 뭐야? ")
print("반가워, " + name + "!")
```

**부모 설명 방법:** "input은 컴퓨터가 우리한테 '뭔가 입력해줘'라고 기다리는 거야"

#### print vs input 차이

| 함수 | 방향 | 역할 |
|------|------|------|
| `print()` | 컴퓨터 → 사람 | 화면에 보여주기 |
| `input()` | 사람 → 컴퓨터 | 키보드 입력 받기 |

#### 숫자 입력 받기

```python
age = input("나이를 입력해: ")    # input은 항상 문자열로 받는다
age_num = int(age)                 # 숫자로 변환
future_age = age_num + 10
print("10년 후엔 " + str(future_age) + "살이야!")
```

또는 한 줄로:

```python
age = int(input("나이를 입력해: "))
```

## 7회차: 조건문 — if/elif/else

**소요 시간:** 60분

### 학습 목표

- `if` 조건문의 구조와 들여쓰기 규칙 이해하기
- `elif`, `else` 의 역할 이해하기
- 조건에 따라 다른 결과가 나오는 프로그램 만들기

### 핵심 개념 설명

#### if — "만약에"

**일상 언어로 먼저:**
"만약 비가 오면 → 우산 챙기기. 아니면 → 그냥 나가기"

```python
weather = "비"

if weather == "비":
    print("우산 챙겨!")
else:
    print("그냥 나가!")
```

#### elif — 여러 조건

```python
score = 85

if score >= 90:
    print("A 등급")
elif score >= 80:
    print("B 등급")
elif score >= 70:
    print("C 등급")
else:
    print("더 열심히!")
```

#### ⚠️ 들여쓰기 — Python의 핵심 규칙

```python
if score >= 90:
    print("A 등급")   # 스페이스 4칸 (또는 Tab 1번)
print("끝")           # 들여쓰기 없음 = if 밖에 있음
```

> **들여쓰기가 빠지면 → IndentationError 발생**
> 이 오류가 나면 먼저 들여쓰기를 확인하는 습관 기르기

#### 비교 연산자 정리

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `==` | 같다 | `age == 11` |
| `!=` | 다르다 | `age != 10` |
| `>` | 크다 | `score > 90` |
| `<` | 작다 | `score < 60` |
| `>=` | 크거나 같다 | `score >= 80` |
| `<=` | 작거나 같다 | `score <= 50` |

### 진행 흐름

#### 도입 (10분)

일상 생활에서 조건 찾기:
"오늘 배가 고프면? / 숙제를 다 했으면? / 비가 오면?"
이것들을 코드로 어떻게 쓸 수 있을지 같이 생각해보기.

#### 개념 설명 (10분)

들여쓰기를 특히 강조. 직접 들여쓰기를 빼서 오류 체험하게 하기.

#### 실습 (30분) — 숫자 맞히기 게임 v1

```python
secret = 7
guess = int(input("1~10 사이 숫자를 맞혀봐: "))

if guess == secret:
    print("정답!")
elif guess < secret:
    print("더 크게!")
else:
    print("더 작게!")

print("정답은 " + str(secret) + " 이었어!")
```

**스스로 도전:**
`elif` 를 추가해서 "많이 틀렸어" / "조금 틀렸어" 구분하기
- 차이가 3 이상: "많이 틀렸어!"
- 차이가 1~2: "거의 다 왔어!"
- 정답: "정답!"

```python
# 도전 힌트 (필요할 때만 보여주기)

else:
    print("많이 틀렸어!")
```

## 8회차: 반복문 — for / while

**소요 시간:** 60분

### 학습 목표

- `for`와 `while` 반복문의 차이 이해하기
- 반복문으로 코드를 훨씬 짧게 쓸 수 있음을 체험하기
- `break`와 `continue`의 역할 이해하기

### 핵심 개념 설명

#### 반복문이 왜 필요한가?

```python
# 반복문 없이 5번 출력
print("Hello")
print("Hello")
print("Hello")
print("Hello")
print("Hello")

# for문으로
for i in range(5):
    print("Hello")
```

**부모 설명 방법:** "똑같은 일을 여러 번 할 때 반복문을 쓰면 코드가 훨씬 짧아져"

#### for문 — 정해진 횟수 반복

```python
for i in range(5):        # 0, 1, 2, 3, 4 (5번)
    print(i)

for i in range(1, 6):     # 1, 2, 3, 4, 5
    print(i)

for i in range(0, 10, 2): # 0, 2, 4, 6, 8 (2씩 증가)
    print(i)
```

#### while문 — 조건이 참인 동안 반복

```python
count = 0
while count < 5:
    print(count)
    count = count + 1   # 또는 count += 1
```

#### for vs while — 언제 쓰나?

| 상황 | 선택 | 이유 |
|------|------|------|
| 몇 번 반복할지 알 때 | `for` | `range(n)` 으로 횟수 지정 |
| 조건이 만족될 때까지 | `while` | 언제 끝날지 모를 때 |
| 무한 반복이 필요할 때 | `while True:` + `break` | 게임 루프 등 |

#### break와 continue

```python
# break — 반복문 즉시 탈출
for i in range(10):
    if i == 5:
        break          # 5에서 멈춤
    print(i)           # 0, 1, 2, 3, 4 출력

# continue — 이번 반복만 건너뜀
for i in range(5):
    if i == 2:
        continue       # 2는 건너뜀
    print(i)           # 0, 1, 3, 4 출력
```

## 9회차: AI와 코드 디버깅 전문 수업

**소요 시간:** 60분

### 학습 목표

- 오류 유형별 원인 파악 방법 익히기
- 오류 메시지를 보고 당황하지 않는 습관 만들기
- 3단계 도움 규칙을 실전으로 적용하기

### 오류 4총사 — 반드시 외워야 할 것들

#### 1. SyntaxError — 문법 오류

```python
# 따옴표 빠짐
print("Hello)       # SyntaxError

# 콜론 빠짐
if age > 10         # SyntaxError (: 가 없음)
    print("ok")
```

**해결:** 빨간 줄이 표시된 위치부터 확인. 오타, 따옴표, 괄호, 콜론 순으로 체크.

#### 2. NameError — 이름 오류

```python
# 변수 이름 오타
nane = "철수"       # 변수를 nane으로 만들었는데
print(name)         # name으로 불러서 NameError

# 정의 전에 사용
print(x)            # x를 아직 만들지 않았는데 사용
x = 10
```

**해결:** 변수 이름 스펠링 확인. 변수를 먼저 만들었는지 확인.

#### 3. IndentationError — 들여쓰기 오류

```python
if age > 10:
print("ok")         # 들여쓰기가 없어서 IndentationError

if age > 10:
        print("ok") # Tab과 Space 혼용해도 IndentationError
```

**해결:** 들여쓰기가 일정한지 확인. Tab과 Space를 혼용하지 않기.

#### 4. TypeError — 타입 오류

```python
age = 11
print("나이: " + age)       # TypeError (문자열 + 숫자 불가)

# 해결
print("나이: " + str(age))  # str()로 변환
```

**해결:** 다른 타입끼리 연산할 때 `str()` 또는 `int()` 로 변환.

#### 왜 함수가 필요한가?

**부모 설명 방법:** "함수는 레시피야. 라면 끓이는 방법을 한번 적어두면, 매번 다시 적지 않고 '라면 끓이기'만 말하면 되잖아"

```python
# 호출

```

```python
print(result)
```

## 12회차: 미니 프로젝트 — 텍스트 게임 완성

**소요 시간:** 90분

### 학습 목표

- 변수·조건문·반복문·함수·리스트를 **종합 사용**하기
- "처음부터 끝까지 혼자 만든 것" 경험하기
- 만든 것을 부모에게 **설명하는 능력** 기르기

### 이 수업의 성공 기준

> 코드의 완성도가 아니라,
> **"왜 이렇게 만들었는지 말로 설명할 수 있는가"** 가 기준.

### 진행 흐름

#### 기획 (15분)

**먼저 AI에게 아이디어 요청 (기획 단계만 허용):**

OpenClaw에 이렇게 질문하기:
"파이썬으로 만들 수 있는 텍스트 RPG 아이디어를 알려줘. 변수, 조건문, 반복문, 함수, 리스트를 다 써야 해. 11살이 만들 수 있는 수준으로."

**기획서를 종이에 먼저 쓰기:**
```
제목: ___________
주인공: ___________
어떤 선택지가 있나?: ___________
어떻게 끝나나?: ___________
어떤 기능이 필요한가?: ___________
```

#### 제작 (60분)

**제작 원칙:**
- AI에게 코드 받지 않기. 기획 아이디어만 OK
- 막히면 3단계 규칙 적용
- 완성보다 이해가 중요. 미완성이어도 괜찮음

**예시 구조 (참고용 — 아이에게 보여주지 않기):**

```python
# 텍스트 RPG 예시 구조
import random

def show_status(hp, items):
    print("\n=== 현재 상태 ===")
    print("HP: " + str(hp))
    print("아이템: " + str(items))

def battle(enemy_name):
    enemy_hp = random.randint(10, 30)
    print(enemy_name + " 등장! (HP: " + str(enemy_hp) + ")")
    return enemy_hp

# 게임 시작
player_name = input("영웅의 이름을 입력해: ")
player_hp = 100
inventory = []

print("\n" + player_name + " 의 모험이 시작됐어!")

rooms = ["숲", "동굴", "마을"]

for room in rooms:
    print("\n=== " + room + " 에 도착했어! ===")
    show_status(player_hp, inventory)

    choice = input("1. 탐험  2. 쉬기: ")

    if choice == "1":
        enemy_hp = battle("고블린")
        damage = random.randint(5, 20)
        player_hp -= damage
        print(str(damage) + " 데미지를 받았어!")

        if player_hp <= 0:
            print("게임 오버!")
            break

        item = "포션"
        inventory.append(item)
        print(item + " 을 획득했어!")

    elif choice == "2":
        heal = 20
        player_hp += heal
        print(str(heal) + " HP 회복!")

print("\n모험 끝! 최종 HP: " + str(player_hp))
```

#### 발표 (15분)

**발표 내용 (아이가 직접 설명):**
1. "이 프로그램은 어떤 프로그램이야?" (1~2문장)
2. "가장 어려웠던 부분은?" + 어떻게 해결했는지
3. "이 코드에서 조건문은 어디에 있어?" → 직접 가리키기
4. "리스트를 어디에 썼어? 왜 리스트를 썼어?"
5. "다음에 추가하고 싶은 기능은?"

**부모 평가 기준:**
- 코드 완성도: 20%
- 설명 능력: 50%
- 스스로 고친 것들: 30%

### 2단계 완료 체크리스트

아이가 스스로 체크:

```

☐ for, while 반복문의 차이를 설명할 수 있다
☐ 함수(def)를 만들고 호출할 수 있다
☐ 리스트에 데이터를 추가·삭제할 수 있다
☐ 오류 메시지를 보고 종류(Syntax/Name/Indent/Type)를 구분할 수 있다
☐ 텍스트 게임을 처음부터 끝까지 혼자 만들었다
☐ 만든 것을 부모에게 설명할 수 있다
```
