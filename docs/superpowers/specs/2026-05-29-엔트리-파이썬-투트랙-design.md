# 설계 문서 — 엔트리·파이썬 투트랙 (2단계)

> **작성일:** 2026-05-29
> **대상 단계:** 2단계 (5~12회차)
> **배경:** 11세 아이가 아직 파이썬 텍스트 코딩에 부담을 느껴, 엔트리(블록코딩)를 주력으로 더 진행하고 파이썬은 가볍게 병행하기로 결정. 1단계 4회차에서 "고양이 피하기" 게임(강사용 선행가이드 MD)을 한 번 만든 경험이 있음.

---

## 1. 목표

- 엔트리를 **주력 트랙**으로, 파이썬을 **맛보기 보조 트랙**으로 회차를 교차하며 진행한다.
- "고양이 피하기" 다음 단계로, **신개념을 점진적으로 얹은 엔트리 게임 3개**와 제작 가이드를 만든다.
- 게임마다 **강사용 HTML(playwright 스크린샷 포함) + 학생용 HTML(과제·힌트만)** 두 자료를 만든다.
- 엔트리에서 익힌 개념(키보드/신호/장면/함수)이 2단계 후반·3단계의 파이썬으로 자연스럽게 이어지게 한다.

## 2. 비목표 (YAGNI)

- 학생용에 정답 블록 코드를 나열하지 않는다 (과제 + 힌트만).
- 드래그로 조립된 블록 결과를 playwright로 자동 캡처하지 않는다 (불안정 — 블록 코드는 HTML 블록 표기로 렌더링).
- 3단계(13~20회차) 자료는 이 작업 범위에 포함하지 않는다.
- 게임 4개 이상으로 확장하지 않는다 (3개로 고정).

---

## 3. 트랙 운영 구조

엔트리 비중을 높이고 회차를 교차한다. 엔트리 게임 1개당 1~2회차, 그 사이에 파이썬을 1회차씩 가볍게 끼운다.

| 회차 | 트랙 | 내용 |
|---|---|---|
| 5 | 🐍 Python | (기존) Pickcode 입문 — Hello / 변수 맛보기 |
| 6~7 | 🧩 Entry ① | **별 받기** — 키보드 조종 + 점수 변수 |
| 8 | 🐍 Python | 조건문 / 입력 가볍게 (엔트리 ①의 조건 개념과 연결) |
| 9~10 | 🧩 Entry ② | **두더지 잡기** — 신호(방송하기) + 변수/타이머 |
| 11 | 🐍 Python | 반복문 가볍게 |
| 12 | 🧩 Entry ③ | **간단 RPG** — 장면 전환 + 함수 |

> 주 1~2회 운영으로 회차는 유동적이다. 아이 속도에 맞춰 엔트리 게임을 늘리거나 파이썬을 줄여도 된다. 위 표는 기본 리듬이며 절대값이 아니다.

## 4. 3개 게임 설계 (개념 점진)

각 게임은 **이전까지 익힌 개념 + 신개념 1~2개**를 얹는다. 피하기 게임이 이미 다룬 개념: 반복(`계속 반복하기`), 복제(`복제본`), 조건(`만일~라면`), 충돌(`~에 닿았는가?`), 변수.

### ① 별 받기 — 신개념: 키보드 입력, 점수 변수

- **한 줄 규칙:** "좌우 키로 바구니를 움직여 위에서 떨어지는 별을 받자. 받으면 점수 +1!"
- **오브젝트:** 바구니/캐릭터(플레이어), 별(복제되어 떨어짐), (선택) 배경
- **핵심 신블록:**
  - `← 키를 눌렀을 때` / `→ 키를 눌렀을 때` (시작) → `x좌표를 ~만큼 바꾸기` (움직임)
  - `점수` 변수 만들기 → `점수에 1 만큼 더하기` (자료)
- **재활용 개념:** 별 복제·낙하(피하기의 폭탄 로직과 동일), `별에 닿았는가?` 충돌 — 단 "닿으면 좋은 것"으로 의미 반전
- **확장 아이디어:** 황금별(점수 +5), 폭탄 섞기(닿으면 -1), 제한시간

### ② 두더지 잡기 — 신개념: 신호 방송하기/받기, 시간 변수

- **한 줄 규칙:** "무작위로 튀어나오는 두더지를 망치로 클릭해서 잡자. 제한시간 안에 몇 마리?"
- **오브젝트:** 두더지(무작위로 나타났다 숨음, 복제 또는 여러 마리), 망치(마우스 따라감), (선택) 배경
- **핵심 신블록:**
  - `신호 만들기` → `~ 신호 보내기` / `~ 신호를 받았을 때` (시작) — 오브젝트끼리 "대화"
  - `타이머 시작하기` / `타이머 값` (계산·판단) 또는 `시간` 변수
  - `모양 보이기` / `모양 숨기기` (생김새) + `~초 기다리기` (흐름)
- **재활용 개념:** 무작위 위치(`무작위 수`), 조건(`만일 망치에 닿고 마우스 클릭했다면`), 점수 변수
- **확장 아이디어:** 두더지 종류별 점수 차등, 점점 빨라지기, 잘못 치면 감점

### ③ 간단 RPG — 신개념: 장면 전환, 함수

- **한 줄 규칙:** "마을에서 출발해 적을 만나면 전투 장면으로! 공격 버튼으로 적을 물리치자."
- **오브젝트/장면:** 주인공, 적, **장면 2개(마을 → 전투)**
- **핵심 신블록:**
  - `장면을 시작하기` / `~ 장면이 시작되었을 때` — 화면 전환
  - `함수 만들기` — 반복되는 "공격" 동작(모양 바꾸기 + 적 체력 감소 + 효과)을 하나의 블록으로 묶기
- **재활용 개념:** 변수(체력/HP), 조건(`만일 HP ≤ 0`), 신호(전투 시작 알림)
- **확장 아이디어:** 회복 아이템, 적 여러 종류, 승리/패배 장면 분기
- **파이썬 다리:** "함수 = 코드 묶음에 이름 붙이기" 개념이 3단계 파이썬 `def`로 직결됨을 강사용에서 명시

## 5. 자료 형식

### 5.1 강사용 HTML (게임당 1개)

- **테마/레이아웃:** 다크(#0f1117), 슬라이드형(전체화면, ← → 이동), 타이머(T키), 진행 바, 슬라이드 카운터 — 기존 강사용 규칙 준수
- **색 계열:** 2단계 Teal(#14b8a6) 유지
- **포함 내용 (기존 `엔트리_선행가이드` 구성 계승):**
  1. 게임 소개 + 한 줄 규칙 + 완성 이미지/목표
  2. 사용할 오브젝트 준비 (스크린샷: 오브젝트 추가/검색 화면)
  3. STEP별 제작 (블록 조립은 **스타일링된 블록 표기**로 렌더링)
  4. 신개념 집중 설명 (이 게임에서 처음 나오는 블록)
  5. 자주 막히는 곳 — 디버깅 체크리스트
  6. 3단계 도움 규칙 적용 (10분 혼자 → AI 힌트 → 말로 설명)
  7. 변형 아이디어
- **스크린샷:** playentry.org 실제 화면. "어디를 누르고 무엇을 검색하는지"를 보여줌. 조립된 코드 자체는 블록 표기로.

### 5.2 학생용 HTML (게임당 1개)

- **테마/레이아웃:** 라이트(#f0f4ff), 탭형(스크롤), 터치 친화(44px+), user-scalable=no — 기존 학생용 규칙 준수
- **포함 내용 — 과제 + 힌트만:**
  - 오늘의 미션(한 줄), 만들 게임 목표
  - 단계별 **과제** ("바구니가 ← → 키로 움직이게 해보자")
  - 단계별 **힌트** (정답 블록 ❌ — "어느 카테고리에서 찾을까?", "키를 눌렀을 때… 무슨 일이 일어나야 하지?" 수준)
  - 체크리스트, 진행률, "이 블록이 뭐 하는 건지 말로 설명하기" 칸
  - 막히면 AI에게 묻는 좋은 질문 예시(학습형 질문)

## 6. 스크린샷 캡처 계획

기존 `scripts/capture-entry.js`(playwright, headless, `playentry.org/ws/` 빈 에디터, 로그인 불필요) 패턴을 확장한다.

- **재사용 (이미 `assets/screenshots/`에 존재):** 홈, 에디터 4영역, 카테고리(시작/흐름/움직임/생김새/판단/계산/자료)
- **신규 캡처 (게임별):**
  - 별 받기: 오브젝트 검색 "별", "바구니"
  - 두더지 잡기: 오브젝트 검색 "두더지", "망치"
  - RPG: 오브젝트 검색(주인공/적), **장면 추가 탭 UI**, **함수 만들기 UI**
- **저장 위치:** `assets/screenshots/entry/<게임>/<순번>_<이름>.png` (게임별 하위 폴더로 정리)
- **블록 조립 결과:** 자동 캡처하지 않음 — HTML 내 블록 표기로 렌더링

## 8. 제작 순서 (권장)

1. **스크린샷 캡처 스크립트** 작성·실행 (게임 ① 기준 먼저) → 화면 확보
2. **게임 ① 별 받기** 강사용 → 학생용 (가장 단순, 패턴 확립)
3. **게임 ② 두더지 잡기** (신호 개념)
4. **게임 ③ RPG** (장면·함수)
5. `2단계_커리큘럼.md` 투트랙 섹션 + `index.html` 링크 갱신

각 게임은 강사용을 먼저 완성해 블록 흐름을 확정한 뒤, 그 미션을 "힌트화"해서 학생용을 만든다.
