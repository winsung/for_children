# 엔트리·파이썬 투트랙 (2단계) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 엔트리 게임 3개(별 받기·두더지 잡기·간단 RPG)의 강사용·학생용 HTML 6개와 스크린샷 캡처 스크립트, 커리큘럼/인덱스 갱신을 만들어 2단계를 엔트리 주력 투트랙으로 전환한다.

**Architecture:** 콘텐츠 제작 프로젝트(단위 테스트 없음). 강사용은 기존 `엔트리_선행가이드.html`(다크·스크롤·섹션·목차형, `.shot`/`.box` 컴포넌트)을 템플릿으로, 학생용은 기존 `2단계_학생용.html`(라이트·탭형, `.checklist`)을 템플릿으로 복제·변형한다. 스크린샷은 기존 `scripts/capture-entry.js`(playwright headless) 패턴을 확장해 `playentry.org/ws/` 빈 에디터에서 로그인 없이 캡처한다. 검증은 브라우저 열람 + 디자인 규칙 grep 체크 + 참조 이미지 존재 확인으로 한다.

**Tech Stack:** 순수 HTML/CSS/JS(인라인, 외부 JS 라이브러리 없음), 구글 폰트 CDN, Node.js + playwright(스크린샷), git.

## 게임별 블록 흐름 (도메인 콘텐츠 — 그대로 렌더링할 것)

> 아래 블록 시퀀스가 강사용 STEP과 학생용 과제·힌트의 원본이다. 구현자는 이 로직을 새로 발명하지 말고 그대로 사용한다. 표기: `[카테고리] 블록내용`, `┣`/`┗`는 들여쓰기(중첩) 표시.

### 게임 ① 별 받기 (신개념: 키보드 입력, 점수 변수)

**오브젝트:** 바구니(플레이어), 별(복제되어 떨어짐), (선택) 배경

**STEP 1 — 오브젝트 준비:** 엔트리봇 삭제 → "바구니" 검색·추가 → "별" 검색·추가. 저장(작품명 "별 받기").

**STEP 2 — 바구니를 키보드로 (바구니 오브젝트 선택):**
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[움직임]   x: 0  y: -110  위치로 이동하기   (화면 하단 중앙)
```
```
[시작]   ← (왼쪽 화살표) 키를 눌렀을 때
[움직임]   x좌표를 -10 만큼 바꾸기
```
```
[시작]   → (오른쪽 화살표) 키를 눌렀을 때
[움직임]   x좌표를 10 만큼 바꾸기
```

**STEP 3 — 점수 변수 만들기:** 자료 → 변수 만들기 → "점수".
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때   (바구니 또는 별, 아무 데나 1번)
[자료]   점수 를 0 (으)로 정하기
```

**STEP 4 — 별이 떨어지기 (별 오브젝트 선택) — 원본:**
```
[시작]   ▶ 시작하기 버튼을 클릭했을 때
[생김새]   모양 숨기기
[흐름]   계속 반복하기
[흐름]     ┣ 자신의 복제본 만들기
[흐름]     ┗ 1 초 기다리기
```

**STEP 5 — 별 복제본 동작 (별 오브젝트, 새 시작 블록):**
```
[시작]   복제본이 처음 생성되었을 때
[생김새]   모양 보이기
[움직임]   x: (-200 부터 200 사이의 무작위 수)  y: 130  위치로 이동하기
[흐름]   계속 반복하기
[움직임]     ┣ y좌표를 -4 만큼 바꾸기
[흐름]     ┣ 만일 ⟨ 바구니 에 닿았는가? ⟩ 라면
[자료]     ┃    ┣ 점수 에 1 만큼 더하기
[흐름]     ┃    ┗ 이 복제본 삭제하기
[흐름]     ┗ 만일 ⟨ y좌표 < -130 ⟩ 라면
[흐름]          ┗ 이 복제본 삭제하기
```

**신개념 포인트:** `키를 눌렀을 때`(키보드 입력) / `점수에 1 만큼 더하기`(변수 누적). 충돌(`닿았는가?`)을 "받으면 좋은 것"으로 재활용.
**자주 막히는 곳:** 바구니가 화면 밖으로 나감(좌우 끝 제한 미설정 — 변형으로 다룸) / 점수 변수가 두 번 초기화됨(시작 블록 중복) / 별이 한 줄로만 떨어짐(무작위 수 누락).
**변형:** 황금별(+5) 추가 / 폭탄 섞기(닿으면 점수 -1) / 제한시간.

### 게임 ② 두더지 잡기 (신개념: 신호 보내기·받았을 때, 시간 변수)

**오브젝트:** 두더지(복제로 무작위 등장), 망치(마우스 따라감), (선택) 배경. **신호:** "잡힘".

**STEP 1 — 오브젝트 준비:** "두더지"(없으면 동물 아무거나) + "망치"(없으면 손/공) 추가. 신호 만들기 → "잡힘". 저장("두더지 잡기").

**STEP 5 — 신호 받기 (배경 또는 점수판 오브젝트 선택) — 신호 받았을 때:**

검색 캡처는 `capture-entry.js` 5~6번 블록(203~236행)과 동일한 input-fill 패턴 사용. 신호/장면/함수 UI는 `page.evaluate`로 해당 버튼 텍스트(`신호 만들기`/`장면 추가`/`함수 만들기`)를 가진 요소를 찾아 `.click()` 후 캡처.

```javascript
// 핵심 골격 (capture-entry.js에서 헬퍼 복사 후)
const GAMES = {
  star: [['바구니','01_search_basket'], ['별','02_search_star']],
  mole: [['두더지','01_search_mole'], ['망치','02_search_hammer']],
};
// 오브젝트 다이얼로그 열기 → 게임별 검색어로 검색 후 shot
async function searchAndShot(page, game, term, name) {
  const inputs = await page.$$('input');
  for (const inp of inputs) {
    if (!(await inp.isVisible())) continue;
    const ph = await inp.getAttribute('placeholder');
    const type = await inp.getAttribute('type');
    if (ph?.includes('검색') || type === 'search' || type === 'text') {
      await inp.fill(''); await inp.fill(term);
      await page.waitForTimeout(2000);
      await shot(page, `entry/${game}/${name}`);
      return;
    }
  }
}
```
