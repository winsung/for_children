1. 게임 소개 — 미로 탈출

🎯 게임 규칙 한 줄

"화살표 키로 주인공을 움직여 벽에 닿지 않고 골에 도착하자. 벽에 닿으면 처음으로!"

사용할 오브젝트

  • 주인공 — 플레이어 (4방향 화살표 키로 움직임)
  • 벽 (미로) — 닿으면 안 되는 길. 그리기 또는 배경/오브젝트로 만든다
  • 골 — 도착 지점 (닿으면 탈출 성공)
🧱 무엇을 쌓아 올리나

이미 만든 별 받기 게임의 키보드 이동과 충돌 판정(닿았는가?)을 그대로 재활용합니다. 이번에는 거기에 '처음 위치로 되돌리기'(벽 충돌 시 시작점 복귀)와 도착 판정(골에 닿으면 성공·정지) 두 가지 새 개념을 얹습니다.

⚠️ 선행 안내 (Kay)

이 문서는 강사용입니다. 아래 블록 배열은 정답입니다 — 수업 전 한번 직접 만들어 보세요. 단, 수업 중에는 자녀에게 정답 블록을 보여주지 말고 AI 도움 규칙(아래)으로 유도하세요.

STEP 1 — 오브젝트 준비

1
엔트리봇 삭제 + 주인공/벽/골 추가 + 저장
⏱ 8분

1-1. 기본 엔트리봇 삭제

좌측 하단 오브젝트 목록에서 "엔트리봇" 위에 마우스 올리기 → X 버튼 클릭.

1-2. 주인공(플레이어) 추가

  1. 오브젝트 목록 위쪽 "+ 오브젝트 추가하기" 클릭
  2. 검색창에 좋아하는 캐릭터(예: "엔트리봇", "고양이", "생쥐")를 입력 → 선택 → 추가하기
🔍 오브젝트 검색 화면 위치

오브젝트 검색·추가 화면은 "+ 오브젝트 추가하기" → 상단 검색창입니다. 미로 전용 캡처 이미지는 없으니, 별 받기/두더지 게임에서 본 것과 동일한 검색 화면이라고 생각하세요. 주인공은 작은 캐릭터(미로 길을 통과할 만한 크기)가 좋습니다.

1-3. 골(도착 지점) 추가

  1. 다시 "+ 오브젝트 추가하기" 클릭
  2. 검색창에 "깃발" 또는 "별", "하트" 등 도착 표시용 오브젝트 입력 → 추가
  3. 골을 미로의 반대쪽 끝(예: 오른쪽 위)에 배치

1-4. 벽(미로) 만들기

두 가지 방법 중 하나를 고르세요.

  • (쉬움) 배경 그리기 — 배경 오브젝트를 추가하고 그리기(붓)로 굵은 선을 그어 미로 길을 만든다. 단, 배경은 "닿았는가?"로 잘 안 잡힐 수 있으니 아래 방법을 권장.
  • (권장) 벽 오브젝트 — "네모"/"사각형" 같은 도형 오브젝트를 여러 개 추가해 길을 막는 벽으로 배치하고, 모두 이름을 "벽"으로 통일하거나 한 오브젝트로 모양을 그린다. "닿았는가?" 드롭다운에서 이 벽을 고른다.
💾 저장

상단 우측 "저장하기" 클릭 → 작품 이름 "미로 탈출". 한번 저장해두면 자동저장도 안전합니다.

엔트리 편집기 영역 (참고)

엔트리 편집기 영역 — 왼쪽 카테고리/블록, 가운데 코드 영역, 위쪽 실행 화면, 아래쪽 오브젝트 목록

블록 카테고리는 어디에? (참고)

아래 STEP에서 쓰는 카테고리 위치입니다. 막혔을 때만 참고하세요.

시작 (시작하기 / 키를 눌렀을 때)
움직임 (위치로 이동하기 / x·y좌표 바꾸기)
흐름 (계속 반복 / 만일~라면 / 모든 코드 멈추기)
판단 (~에 닿았는가?)
🔍 블록 못 찾을 때

블록 모음 위쪽 검색창에 일부만 입력해도 찾아줍니다. 예: "키", "닿", "위치", "이동", "반복", "멈추기".

STEP 2 — 주인공 시작 위치

2
시작할 때 주인공을 미로 입구(시작점)에 두기
⏱ 6분

⚠️ 오브젝트 목록에서 "주인공" 클릭 (반드시 먼저!)

2-1. 시작할 때 주인공을 시작점으로

[시작] ▶ 시작하기 버튼을 클릭했을 때 [움직임] x: -200 y: -100 위치로 이동하기 // 미로 입구(시작점)
🎯 핵심 트릭

이 시작점 좌표 x: -200 y: -100는 STEP 4(벽에 닿으면 처음으로)에서 똑같이 다시 사용됩니다. 그래서 두 곳의 좌표가 반드시 같아야 합니다. 미로 모양에 맞게 좌표를 바꾸면 STEP 4도 같이 바꾸세요.

✅ 테스트

▶ 시작하기 → 주인공이 항상 미로 입구(왼쪽 아래)에서 출발하는지 확인합니다.

STEP 3 — 화살표 4방향 이동

3
↑↓←→ 4개의 키 흐름 (주인공)
⏱ 12분

⚠️ 같은 "주인공" 오브젝트에 화살표 키마다 새 시작 블록을 4개 만듭니다.

3-1. 위쪽 화살표 → 위로

[시작] ↑ (위쪽 화살표) 키를 눌렀을 때 [움직임] y좌표를 5 만큼 바꾸기

3-2. 아래쪽 화살표 → 아래로

[시작] ↓ (아래쪽 화살표) 키를 눌렀을 때 [움직임] y좌표를 -5 만큼 바꾸기

3-3. 왼쪽 화살표 → 왼쪽으로

[시작] ← (왼쪽 화살표) 키를 눌렀을 때 [움직임] x좌표를 -5 만큼 바꾸기

3-4. 오른쪽 화살표 → 오른쪽으로

[시작] → (오른쪽 화살표) 키를 눌렀을 때 [움직임] x좌표를 5 만큼 바꾸기
🎯 핵심 트릭

~ 키를 눌렀을 때 블록은 시작 카테고리에 있고, 드롭다운에서 "위쪽/아래쪽/왼쪽/오른쪽 화살표"를 골라야 합니다. 별 받기에서는 좌우만 썼지만, 미로는 4방향 모두 필요합니다. 한 칸 이동을 5로 둔 이유: 너무 크면 벽을 뚫고 지나가 충돌 판정이 안 잡힙니다.

✅ 테스트

▶ 시작하기 → 실행 화면을 한 번 클릭(키 입력 활성화) → 화살표 4방향으로 주인공이 움직이는지 확인합니다.

STEP 4 — 벽에 닿으면 처음으로

4
계속 확인하다 벽에 닿으면 시작점으로 되돌리기 (주인공)
⏱ 12분

⚠️ 같은 "주인공" 오브젝트에 새 시작 블록으로 별도 흐름을 추가합니다.

[시작] ▶ 시작하기 버튼을 클릭했을 때 [흐름] 계속 반복하기 [흐름] ┗ 만일 ⟨ 벽 에 닿았는가? ⟩ 라면 [생김새] ┣ "처음부터!" 을(를) 말하기 [움직임] ┗ x: -200 y: -100 위치로 이동하기 // STEP 2와 같은 시작점
  • 벽 에 닿았는가? → 판단 카테고리 (드롭다운을 "벽"으로 변경)
  • "처음부터!" 을(를) 말하기 → 생김새 카테고리
  • x: -200 y: -100 위치로 이동하기 → 움직임 카테고리 (STEP 2와 좌표가 같아야 함!)
🆕 신개념 — 처음 위치로 되돌리기

이것이 이 게임의 핵심입니다. "계속 반복"으로 매 순간 벽 충돌을 감시하다가, 닿는 순간 시작점 좌표로 순간이동시켜 처음부터 다시 하게 만듭니다. 별 받기에서 "닿으면 삭제"였던 충돌 처리가, 여기서는 "닿으면 원위치"로 바뀝니다.

✅ 체크포인트

▶ 시작하기 → 일부러 벽으로 주인공을 밀어보세요. 닿는 순간 "처음부터!"를 말하며 시작점으로 돌아가면 성공입니다.

STEP 5 — 골 도착 판정

5
골에 닿으면 "탈출 성공!" + 모든 코드 멈추기 (주인공)
⏱ 8분

⚠️ 같은 "주인공" 오브젝트에 또 하나의 새 시작 블록으로 추가합니다.

[시작] ▶ 시작하기 버튼을 클릭했을 때 [흐름] 계속 반복하기 [흐름] ┗ 만일 ⟨ 골 에 닿았는가? ⟩ 라면 [생김새] ┣ "탈출 성공!" 을(를) 말하기 [흐름] ┗ 모든 코드 멈추기
  • 골 에 닿았는가? → 판단 카테고리 (드롭다운을 "골"로 변경)
  • "탈출 성공!" 을(를) 말하기 → 생김새 카테고리
  • 모든 코드 멈추기 → 흐름 카테고리
🎯 핵심 트릭

닿았는가?는 처음에 "마우스포인터에 닿았는가?"로 나옵니다. 드롭다운을 "골"로 변경해야 합니다. 모든 코드 멈추기를 넣어야 골에 도착한 뒤 STEP 4의 "벽 감시"까지 멈춰 게임이 깔끔하게 끝납니다.

✅ 완성 테스트

▶ 시작하기 → 벽을 피해 골까지 이동 → 골에 닿으면 "탈출 성공!"을 말하고 모든 움직임이 멈추면 완성입니다.

2. 신개념 포인트

🆕 이번에 새로 배우는 것

① 처음 위치로 되돌리기 — 벽에 닿으면 x: -200 y: -100 위치로 이동하기로 시작점에 순간이동. "실패하면 처음부터"라는 게임 규칙을 좌표 한 줄로 구현합니다. (시작 위치와 복귀 위치가 같은 값이라는 점이 포인트)
② 도착 판정 — 골 에 닿았는가?가 참이면 모든 코드 멈추기로 게임 종료. "성공 조건을 감시하다 만족하면 멈춘다"는 패턴입니다.
③ 재활용 — 키보드 이동(키를 눌렀을 때)과 충돌 판정(닿았는가?)은 별 받기 게임에서 이미 배운 것을 그대로 가져왔습니다. 같은 블록을 "닿으면 원위치 / 닿으면 성공"이라는 새 목적에 쓰는 것이 핵심 통찰입니다.

3. 자주 막히는 곳

⚠️ 디버깅 체크리스트

벽을 통과해 버림 → 한 번에 이동하는 칸(5)이 너무 크거나, "닿았는가?" 드롭다운이 "벽"이 아닌 다른 걸 가리킴. 이동 값을 줄이거나 드롭다운을 확인.
벽에 닿아도 안 돌아감 → STEP 4의 계속 반복하기를 빠뜨려 한 번만 검사됨. 충돌 감시는 계속 반복 안에 있어야 함.
되돌아가는데 시작점이 이상함 → STEP 2의 시작 좌표와 STEP 4의 복귀 좌표가 다름. 두 값(-200, -100)을 똑같이 맞추세요.
골에 닿았는데 안 멈춤 → 모든 코드 멈추기를 빠뜨렸거나, "닿았는가?" 드롭다운이 "골"이 아님.
키를 눌러도 안 움직임 → 실행 화면을 한 번 클릭해 키 입력을 활성화했는지, 오브젝트가 "주인공"으로 선택돼 있는지 확인.
주인공이 벽에 끼어 못 빠져나감 → 시작점이 벽과 너무 가까움. 시작점을 벽에서 살짝 떨어뜨리거나 미로 입구를 넓히세요.

4. AI 도움 규칙 (3단계)

💬 이 프로젝트의 핵심 원칙

(1) 10분 혼자 생각 → (2) AI에게 힌트만 요청 → (3) 직접 수정 + 말로 설명

자녀가 막혀도 즉답하지 않기. "어디서 막혔는지 말로 설명해봐" → "그럼 어떤 블록이 필요할 것 같아?" 순서로 유도하세요.

  • 좋은 질문 예: "엔트리에서 벽에 닿으면 캐릭터를 시작 자리로 되돌리려면 어떤 블록이 필요해? 정답 말고 힌트만."
  • 나쁜 질문 예: "미로 탈출 게임 코드 알려줘."
🎯 핵심 기준

완성 후 "이 블록이 왜 이렇게 동작하는지 말로 설명할 수 있는가?"를 꼭 확인하세요. 특히 "벽에 닿으면 왜 좌표를 시작점으로 다시 옮기는지", "골에 닿았을 때 왜 모든 코드를 멈추는지" 설명하게 해보세요.

5. 변형 아이디어

자녀가 빨리 끝나거나 더 만들고 싶어할 때:

  • 타이머 추가 — "시간" 변수를 만들어 1초마다 +1, 골 도착 시 걸린 시간을 말하기로 보여주기.
  • 목숨(라이프) — "목숨" 변수를 3으로 시작, 벽에 닿을 때마다 -1, 0이 되면 모든 코드 멈추기로 게임 오버.
  • 아이템 모으기 — 미로 곳곳에 별/코인 오브젝트를 두고, 닿으면 점수 +1 후 숨기기. 별 받기에서 배운 점수 변수 재활용.
  • 미로 2단계 — 골에 닿으면 다음 미로(다른 배경)로 바뀌게 만들기. RPG에서 배운 장면 활용.

이런 변형도 AI에게 "어떤 블록을 추가해야 할까?"로 묻게 유도하세요.
