🧠 AI 학습 방식 설명 가이드
AI는 엄청나게 많은 글을 읽고 "패턴"을 기억한 거야.
질문을 받으면 그 패턴으로 가장 어울리는 다음 단어를 만들어내.
인터넷에서 실시간으로 찾아보는 건 아니야.

비유: "수천 권의 책을 읽은 뒤 시험 볼 때 기억나는 대로 쓰는 것"
→ 그래서 가끔 그럴듯하지만 틀린 말도 해! (환각 현상)
📝 참고 — 존댓말과 프롬프팅
존댓말로 질문하면 결과가 더 좋다는 이야기가 있는데,
존댓말 자체의 효과라기보다 존댓말로 쓰면 문장이 자연스럽게 더 정돈되고 구체적이 되기 때문.
핵심은 말투가 아니라 질문의 명확성이라는 점을 아이에게 전달.
💡 부모 팁
"AI가 다 해주면 되잖아요" → "AI한테 공원 놀러가달라고 해봐. 할 수 있을까?"

실험 4 — 환각(Hallucination) 체험
AI가 자신감 있지만 틀린 답을 하는 경우를 직접 확인!
// 이 질문을 해봐 "동그란 삼각형이 뭐야?" // 또는 "바나나가 파란색인 이유를 알려줘"

AI는 패턴으로 답을 만들어내기 때문에,
말이 안 되는 질문에도 "있는 척" 답할 수 있다.

⏱ 10분
화면 4개 영역
①
실행 화면 (장면) — 좌측 상단. ▶ 시작하기 누르면 게임이 여기서 돌아감
②
오브젝트 목록 — 좌측 하단. 캐릭터·배경·장애물 모두 여기에 들어감
③
블록 모음 — 중앙. 시작/흐름/움직임/생김새/판단/계산/자료 카테고리
④
블록 조립소 — 우측. ③에서 블록을 드래그해서 붙이는 코드 영역
기본 동작 3가지
블록 조립: ③ → ④로 끌면 자석처럼 위아래 붙음
블록 분리: 잡고 빈 곳으로 끌면 떨어짐, 휴지통에 끌면 삭제
오브젝트 전환: ②에서 오브젝트 클릭 → ④의 코드가 그 오브젝트 코드로 바뀜
⚠️ 가장 흔한 실수 #1
다른 오브젝트에 코드 짜기

고양이 코드를 짜는 줄 알았는데 폭탄에 붙어있는 경우.

해결: 블록 추가 전 항상 ② 좌측 하단에서 "어떤 오브젝트가 선택돼 있나?" 먼저 확인.
💡 도입 멘트 예시
"여기 ▶ 누르면 게임 시작이고, 여기 사각형이 게임 화면이야.
네가 만들 캐릭터들은 여기 목록에 들어가고,
저쪽 블록들을 끌어다 붙이면 캐릭터가 움직여."
📝 첫 저장
상단 우측 "저장하기" → 작품 이름 "고양이 피하기"
한번 저장해두면 자동저장도 안전함.
STEP 1~2
오브젝트 준비 + 캐릭터 마우스 따라가기
첫 성공 경험을 빨리 만든다
⏱ 15분
STEP 1 — 오브젝트 추가 (5분)
① 기본 엔트리봇 삭제 (오브젝트 위 X 버튼)
② + 오브젝트 추가하기 → "동물" → 고양이 선택 → 추가
③ 다시 + 오브젝트 추가하기 → 검색 "폭탄" 또는 "운석" → 추가
④ (선택) 배경 추가
STEP 2 — 고양이가 마우스를 따라가게 (10분)
② 오브젝트 목록에서 "고양이" 클릭 (반드시 먼저!)
// 고양이 오브젝트의 코드 ▶ 시작하기 버튼을 클릭했을 때 계속 반복하기 { 마우스포인터 위치로 이동하기 }
테스트: ▶ 시작하기 → 마우스 움직이면 고양이가 따라옴 ✅
🎯 이 단계 핵심
"내가 짠 코드가 화면에서 움직였다"는 첫 성공 경험.
여기서 막히면 진도 못 나감 → 빨리 통과시키기.
⚠️ 자주 막히는 곳
고양이가 안 움직임 → ② 오브젝트가 "고양이"로 선택돼 있는지 확인
"마우스포인터" 블록 못 찾음 → "움직임" 카테고리에서. "x: y: 위치로" 와 다른 블록!
블록이 안 붙음 → 자석처럼 붙는 위치까지 살짝 더 끌어가기
💬 강사 멘트
"계속 반복하기는 게임이 끝날 때까지 안에 있는 걸 무한히 한다는 뜻이야.
그래서 마우스를 움직일 때마다 따라오는 거지."
STEP 3 ⚠️ 가장 어려움
폭탄이 계속 복제되며 위에서 떨어지기
복제본 개념 + 무작위 위치 + 화면 끝 처리
⏱ 25분
3-1. 폭탄 본체 — 1초마다 복제
② 오브젝트 목록에서 "폭탄" 클릭
▶ 시작하기 버튼을 클릭했을 때 x: 0 y: 200 위치로 이동 모양 숨기기 // 원본은 안 보이게 계속 반복하기 { 자신의 복제본 만들기 1 초 기다리기 }
3-2. 복제본 — 떨어지기
같은 폭탄 코드 영역에 새 시작 블록으로 따로 추가
복제본이 처음 생성되었을 때 모양 보이기 x: (-200~200 무작위 수) y: 200 위치로 이동 계속 반복하기 { y좌표를 -5 만큼 바꾸기 만일 ⟨ y좌표 < -130 ⟩ 라면 { 이 복제본 삭제하기 } }
🧠 복제본 개념 설명
"폭탄 하나만으로 여러 개를 만들려면 어떻게 할까?
→ 복사본을 계속 만들어내는 거야.
원본은 화면 밖에 숨어있고, 복사본만 떨어져."
⚠️ 자주 막히는 곳
한 자리에서만 떨어짐 → "무작위 수" 블록을 시작 x에 안 끼움
화면이 점점 느려짐 → "이 복제본 삭제하기" 누락
원본도 같이 떨어짐 → "모양 숨기기" 빠뜨림
아무것도 안 떨어짐 → 두 시작 블록 중 하나가 다른 오브젝트에 붙어있음
💡 막히면 AI에게 (힌트만)
"엔트리에서 오브젝트가 화면 위에서 무작위 위치로 계속 떨어지게 하려면 어떤 블록을 조합해야 해? 정답 코드 말고 어떤 블록이 필요한지만 알려줘."
✅ 체크포인트
▶ 시작하기 → 폭탄들이 무작위 위치에서 떨어지고, 바닥에 닿으면 사라짐. 화면이 안 느려짐.
STEP 4
폭탄에 닿으면 게임 오버
조건문 등장 — "만약 ~라면"
⏱ 15분
충돌 감지 코드
② "고양이" 다시 클릭 — STEP 2 코드는 그대로 두고, 새 시작 블록으로 추가
// 고양이의 두 번째 시작 흐름 ▶ 시작하기 버튼을 클릭했을 때 계속 반복하기 { 만일 ⟨ 폭탄에 닿았는가? ⟩ 라면 { "게임 오버!" 을(를) 말하기 모든 코드 멈추기 } }
핵심: "닿았는가?" 블록은 처음에 "마우스포인터에 닿았는가?"로 나옴 → 드롭다운을 "폭탄"으로 변경
테스트
▶ 시작하기 → 일부러 폭탄에 부딪쳐보기
→ "게임 오버!" 말풍선 + 모든 움직임 정지 ✅
🧠 조건문 비유
"만약 ~ 라면은 너희가 일상에서 늘 쓰는 말이야.
'만약 비가 오면 우산 쓰자.'
코드도 똑같아 — 어떤 일이 생기면 다음 동작을 하라는 명령."
⚠️ 자주 막히는 곳
충돌해도 게임 안 끝남 → "닿았는가?" 드롭다운이 "마우스포인터" 그대로
"모든 코드 멈추기" 못 찾음 → "흐름" 카테고리 맨 아래쪽
말풍선 안 뜸 → "말하기" 블록의 입력칸이 비어있음
💬 강사 질문
"왜 계속 반복 안에 만약을 넣었을까?"
→ "한 번만 검사하면 충돌 시점을 놓치니까!" 끌어내기
STEP 5 · 변형
시간이 남으면 — 점수/속도/목숨 변형
AI에게 "힌트만" 요청하는 연습
⏱ 10분
기본: 버틴 시간 변수
자료 카테고리 → 변수 만들기 → "버틴 시간"
// 고양이에 추가 ▶ 시작하기 버튼을 클릭했을 때 버틴 시간을 0으로 정하기 계속 반복하기 { 버틴 시간에 1 만큼 더하기 1 초 기다리기 }
화면 좌측 상단에 변수 박스 자동 표시 ✅
변형 아이디어 (아이가 고름)
A
별 모으기 모드 — 폭탄 대신 별, 닿으면 +1점 + 복제본 삭제
B
속도 점점 빨라지기 — 시간 ↑ 일수록 폭탄 떨어지는 간격 ↓
C
목숨 시스템 — 변수 "목숨"=3, 닿으면 -1, 0이 되면 게임 오버
D
두 종류 장애물 — 폭탄 + 운석 두 오브젝트 함께 떨어짐
💡 AI에게 묻는 좋은 예
"엔트리에서 변수를 만들어서 점점 빨라지게 하려면 어떤 블록이 필요해? 코드 말고 블록 이름만."
